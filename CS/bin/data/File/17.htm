第17章  使用数据处理类控件

数据处理类控件实现了数据库编程接口，可以让我们使用很少的代码就可以实现基本的数据库操作。另外，利用数据库处理控件也可以实现丰富的数据处理界面，更加直观方便地让用户对数据进行各种操作，包括增删改查等。本章介绍常用的几种数据处理控件，它们都与数据库操作有关。

17.1  数据控件概述

在VS.NET中，有一个新增的数据控件DataGridView，具备非常丰富的数据处理能力和显示能力，是数据库编程过程中使用最广泛的控件。另外还有与之配合的几个数据库控件，它们组合在一起，就可以形成功能很好的数据库处理程序。

² DataGridView 控件：DataGridView 控件提供用来显示数据的可自定义表。使用 DataGridView 类，可以自定义单元格、行、列和边框。

² BindingSource 组件:通过提供当前项管理、更改通知和其他服务，来简化将窗体上的控件绑定到数据的过程。

² BindingNavigator 控件:提供工具栏式的界面来定位和操作窗体上的数据。

17.1.1  DataGridView控件

DataGridView 控件提供一种强大而灵活的以表格形式显示数据的方式。可以使用 DataGridView 控件来显示少量数据的只读视图，也可以对其进行缩放以显示特大数据集的可编辑视图。

可以用很多方式扩展 DataGridView 控件，以便将自定义行为内置在应用程序中。例如，可以采用编程方式指定自己的排序算法，以及创建自己的单元格类型。通过选择一些属性，可以轻松地自定义 DataGridView 控件的外观。可以将许多类型的数据存储区用作数据源，也可以在没有绑定数据源的情况下操作 DataGridView 控件。

默认情况下，DataGridView 控件具有下列特点：

² 自动显示垂直滚动表时保持可见的列标头和行标头。

² 拥有行标头，其中包含当前行的选中指示符。

² 在第一个单元格中拥有选择矩形。[雨林木风1] 

² 拥有列，当用户双击列分隔符时可自动调整大小。

² 通过应用程序的 Main 方法调用 EnableVisualStyles 方法时，自动支持 Windows XP 和 Windows Server 2003 系列中的视觉样式。

此外，默认情况下可以编辑 DataGridView 控件的内容：

² 用户在某个单元格中双击或按 F2 时，此控件将自动使该单元格自动进入编辑模式，并在用户键入时自动更新单元格的内容。

² 如果用户滚动至网格的结尾，将会看到用于添加新记录的行。用户单击此行时，会向 DataGridView 控件添加使用默认值的新行。用户按 Esc 时，此新行将消失。

² 如果用户单击行标头，将会选中整行。

² 通过设置 DataGridView 控件的 DataSource 属性将其绑定到数据源时，该控件可以：[雨林木风2] 

² 将数据源列的名称自动用作列标头文本。

² 用数据源的内容进行填充。DataGridView 列是为数据源中的每个列自动创建的。

² 为表中的每个可见行创建一行。

² 用户单击列标头时，将根据基础数据自动对行进行排序。

17.1.2  BindingSource 组件

BindingSource 组件作为窗体上部分或全部组件的数据源。在 Visual Studio 中，可以通过 DataBindings 属性将 BindingSource 绑定到控件，该属性可在 属性”窗口中访问。

可以将 BindingSource 组件绑定到两种数据源：

² 简单数据源，如对象的单个属性或 ArrayList 这样的基本集合。

² 复杂数据源，如数据库表。BindingSource 组件作为一个媒介提供绑定和货币管理服务。

在设计或运行时，通过将 BindingSource 组件的 DataSource 和 DataMember 属性分别设置为数据库和表，可以将该组件绑定到复杂数据源。图17-1说明在现有的数据绑定结构中，BindingSource 组件适合放在何处。

注意：在设计时，某些操作（如将数据库表从数据窗口拖动到空白窗体）将创建 BindingSource 组件，将该组件绑定到基础数据源，然后添加数据识别控件整体操作。

如果没有首先指定要绑定到的列表就开始将项添加到 BindingSource 组件，该组件将被用作列表样式的数据源，并接受添加的这些项。

另外，还可以编写代码，通过 AddingNew 事件提供自定义 AddNew功能，当 AddNew 方法在项被添加到列表之前被调用时会引发该事件。

对于需要在窗体上定位数据的用户，BindingNavigator 组件与 BindingSource 组件配合使您能够定位和操作数据。

17.3  在数据库中存储图像

要把图像存储到数据库，可以使用数据库的二进制字段来存储，保证图像的原始格式不发生变化即可。这里利用带有参数的 insret 语句来完成图像文件数据保存到数据库字段的工作。

17.3.1  存储原理

显示图像可以采用图像控件来完成。利用该控件的 FromStream 方法，可以直接从内存数据流中（MemoryStream）对象中获取图像数据并显示。

MemoryStream 类创建这样的流，该流以内存而不是磁盘或网络连接作为支持存储区。MemoryStream 封装以无符号字节数组形式存储的数据，该数组在创建 MemoryStream 对象时被初始化，或者该数组可创建为空数组。可在内存中直接访问这些封装的数据。内存流可降低应用程序中对临时缓冲区和临时文件的需要。

GetBuffer 方法返回从其创建此流的无符号字节数组。

流的 current position 是下一个读取或写入操作可能发生的位置。当前位置可以通过 Seek 方法检索或设置。在创建 MemoryStream 的新实例时，当前位置设置为零。

用无符号字节数组创建的内存流提供无法调整大小的数据流视图，而且只能向其写入。当使用字节数组时，虽然根据传递到构造函数中的参数可能能够修改现有内容，但既不能追加也不能收缩流。空内存流是可调整大小的，而且可以向其写入和从中读取。

为了及时提示用户当前数据库连接状态，专门设计了一个窗体，用来显示信息，窗体上只有一个现实文本的标签控件，然后添加一个Show方法，专门显示信息，Show方法的代码是：

public void Show(string Message) //显示的信息作为参数

  {

    lblStatus.Text = Message;//给窗体的标签控件赋值

   this.Show();//显示窗体自身

   Application.DoEvents();

}

其中使用了Application对象的DoEnvents方法。当运行 Windows 窗体时，它将创建新窗体，然后该窗体等待处理事件。该窗体在每次处理事件时，均将处理与该事件关联的所有代码。所有其他事件在队列中等待。在代码处理事件时，应用程序并不响应。例如，当将另一窗口显示到该窗口前面时，该窗口不重新显示信息。

如果在代码中调用 DoEvents，则应用程序可以处理其他事件。例如，如果代码中有向 ListBox 添加数据的窗体，并将 DoEvents 添加到代码中，那么当将另一窗口显示到当前窗体上时，该窗体将重新显示信息。如果从代码中移除 DoEvents，那么在按钮的单击事件处理程序执行结束以前，窗体不会重新显示信息。
