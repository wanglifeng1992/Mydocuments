第10章  泛型、迭代器和集合

我们在编程的时候，经常会遇到数据类型不同的情况，处理的方法往往是进行数据类型转换。而对于动态传递的数据来说，如果要控制数据类型的一致性，就要不断判断类型，并根据类型来转换，确保数据运算中的类型一致性。这些处理过程是非常麻烦的，C#中提供了泛型技术，可以根本上解决这些问题。

迭代器作为访问对象的特殊方法，具有更加高效和方便的功能，能够实现类的循环迭代访问。本章将说明泛型和迭代器的原理及使用。

10.1  泛型

泛型是 2.0 版 C# 语言和公共语言运行库 (CLR) 中的一个新功能。泛型将类型参数的概念引入 .NET Framework。例如，通过使用泛型类型参数 T，可以编写其他客户端代码能够使用的单个类，而不致引入运行时强制转换或装箱操作的成本或风险。本节介绍泛型接口技术。

10.1.1  泛型概述

泛型是可适应对多种数据类型执行相同功能的单个编程元素。定义泛型类或方法时，无需为可能需要对其执行该功能的每个数据类型定义单独版本。

就好比是带有可拆卸刀头的螺丝刀。我们检查需要拧动的螺丝，然后选择适合该螺丝的刀头（一字、十字、星形）。将正确的刀头插入到螺丝刀柄上后，就可以使用螺丝刀执行完全相同的功能，即拧螺丝。

图10-1  泛型原理

定义泛型类型时，即使用一个或多个数据类型将其参数化。这样可允许使用代码定制数据类型以满足其要求。代码可以通过泛型元素声明若干个不同的编程元素，每个元素可使用一组不同的数据类型。但是，无论声明的元素使用哪些数据类型，它们均执行相同的逻辑。

.NET Framework 类库在 System.Collections.Generic 命名空间中包含几个泛型集合类。如 System.Collections 命名空间中的 ArrayList。

创建泛型类的过程为：从一个现有的具体类开始，逐一将每个类型更改为类型参数，直至达到通用化和可用性的最佳平衡。

创建自己的泛型类时，需要特别注意以下事项：

² 将哪些类型通用化为类型参数。一般规则是，能够参数化的类型越多，代码就会变得越灵活，重用性就越好。

² 是否将泛型行为分解为基类和子类。由于泛型类可以作为基类使用，此处适用的设计注意事项与非泛型类相同。

² 是否实现一个或多个泛型接口。例如，如果设计一个类，该类将用于创建基于泛型的集合中的项，则可能需要实现一个接口，如 IComparable<T>，其中 T 是类的类型。

System.Collections.Generic 命名空间包含定义泛型集合的接口和类，泛型集合允许用户创建强类型集合，它能提供比非泛型强类型集合更好的类型安全性和性能。

10.1.2  泛型的实现

为了体现泛型接口的作用，首先我们看如果不使用泛型进行编程，将会遇到什么问题。我们假设要设计一个Stack类，实现数据堆栈功能，完成数据的入栈和出栈操作。简单的方法是使用一个数组来存储数据。下面是类的代码。

【例10-1-1】实现一个堆栈。

（代码位置：光盘\源代码\chapter10\stack）

namespace stack

{

    class Stack

    {

        private object[] m_ItemsArray;

        private int m_Index = 0;

        public const int MAX_SIZE = 100;

        //初始化堆栈

        public Stack()

        {

            m_ItemsArray = new object[MAX_SIZE];

        }

        //出栈处理

        public object Pop()

        {

            if (m_Index == 0)

                throw new System.InvalidOperationException(

                   "不能从空栈弹出数据。");

            return m_ItemsArray[--m_Index];

        }

        //入栈处理

        public void Push(object item)

        {

            if (m_Index == MAX_SIZE)

                throw new System.StackOverflowException( "不能在满栈中添加数据。");

            m_ItemsArray[m_Index++] = item;

        }

    }

}

在以上代码的第5行中，定义了一个对象数组来存储数据对象。但要注意，并没有明确定义数据对象的类型，可能是各种类型。因此，要处理堆栈中的数据，就要明确区分到底是什么类型的数据。比如下面的代码向堆栈加入了整数数据，则取出的时候就要转换类型：

Stack stack = new Stack();

stack.Push(1234);

int number = (int)stack.Pop();

当我们使用值类型的元素的时候，隐含执行了装箱和拆箱操作。另外，在同样的类的实例中存储不同的类型的元素，这会引发数据类型转换异常错误。比如下面的代码：

Stack stack = new Stack();

stack.Push("1234");//字符串数据元素

int number = (int)stack.Pop();    // 出现异常 InvalidCastException.

异常信息提示我们出现了类型问题，这显然会导致运行时候的代码安全性。我们需要编写代码来控制这种异常的出现。这就意味着，我们必须确保所有可能的类型转换问题都被捕获并加以处理，显然这是很繁琐的。

要实现数据类型的安全处理是可行的。实际上，下面的类StackOfInt 专门处理整数堆栈，而类StackOfSring专门用来处理字符串数据堆栈，以此类推，可以针对具体的数据类型编写不同的类。

C#提供的泛型技术可以解决以上问题。可以通过声明T类型来代表各种数据类型，无论实例化过程中具体数据类型是什么，都使用T来代替。下面的代码使用T来表示堆栈数据类型。

2.  namespace t_stack

3.  {

4.  class Stack<T>

5.  {

7.  private int m_Index = 0;

8.  public const int MAX_SIZE = 100;

9.  public Stack() { m_ItemsArray = new T[MAX_SIZE]; }

10.  public T Pop()

11.  {

12.  if (m_Index == 0)

13.  throw new System.InvalidOperationException("无法弹出空栈。");

14.  return m_ItemsArray[--m_Index];

15.  }

16.  public void Push(T item)

17.  {

18.  if (m_Index == MAX_SIZE)

19.  throw new System.StackOverflowException("堆栈已满，无法入栈。");

20.  m_ItemsArray[m_Index++] = item;

21.  }

22.  }

23.  class Program

24.  {

25.  static void Main()

26.  {

² 无需进行类型转换。

² 不要求装箱和拆箱操作。

² 代码的类型安全。无需在类中控制类型转换，或者为不同的类型编写不同的类。只要在实例化阶段明确即可。

在Stack<T> 中的“T”是类的类型参数。用于参数的多态处理。实际上，Stack<T> 类可以代表多种形式：Stack<int>、Stack<string> 等。Stack<T> 代表各种可能的堆栈操作，并实现了堆栈处理的各种方法。

10.2  迭代器

在软件的构建过程中,集合对象内部结构常常变化各异。但对于这些集合对象,我们希望在不暴露其内部结构的同时，可以让外部客户代码透明地访问其中包含的元素；同时这种“透明遍历”也为“同一种算法在多种集合对象上进行操作”提供了可能。使用面向对象技术将这种遍历机制抽象为“迭代器对象”为“应对变化中的集合对象”提供了一种优雅的方式。本节介绍C#迭代器的实现。

10.2.1  迭代器概述

迭代器是方法、get 访问器或运算符，它使您能够在类或结构中支持 foreach 迭代，而不必实现整个 IEnumerable 接口。只需提供一个迭代器，即可遍历类中的数据结构。当编译器检测到迭代器时，它将自动生成 IEnumerable 或 IEnumerable 接口的 Current、MoveNext 和 Dispose 方法。具体特点是：

² 迭代器是可以返回相同类型的值的有序序列的一段代码。

² 迭代器可用作方法、运算符或 get 访问器的代码体。

² 迭代器代码使用 yield return 语句依次返回每个元素。

² yield break 将终止迭代。

可以在类中实现多个迭代器。每个迭代器都必须像任何类成员一样有唯一的名称，并且可以在 foreach 语句中被客户端代码调用，如下所示：

foreach(int x in SampleClass.Iterator2){}

迭代器的返回类型必须为 IEnumerable、IEnumerator、IEnumerable 或 IEnumerator。

yield 关键字用于指定返回的值。到达 yield return 语句时，会保存当前位置。下次调用迭代器时将从此位置重新开始执行。

迭代器对集合类特别有用，它提供一种简单的方法来迭代不常用的数据结构（如二进制树）。

创建迭代器最常用的方法是对 IEnumerable 接口实现 GetEnumerator 方法，例如：

public System.Collections.IEnumerator GetEnumerator()

{

    for (int i = 0; i < max; i++)

    {

        yield return i;

    }

}

GetEnumerator 方法的存在使得类型成为可枚举的类型，并允许使用 foreach 语句遍历类中的成员。

10.2.2  迭代器的实现

本节通过实例来说明迭代器的实际使用。可以使用循环来遍历结构类型的数据，比如数组和集合对象。例如：

 

string[] cities = {"北京","上海","广州"};

foreach(string city in cities)

{

    Console.WriteLine(city);

}

只要我们实现了GetEnumerator方法并返回Ienumerator接口，就可以实现任何数据集合的遍历。常见的实现代码如下：

public interface IEnumerable

{

    IEnumerator GetEnumerator();

}

//定义接口，实现遍历方法

public interface IEnumerator

{

    object Current { get;}

    bool MoveNext();

    void Reset();

}

有多种方式可以实现迭代器。首先要创建一个“可枚举的类”，这个类实现IEnumerable 接口，可以循环遍历数据集合。例如：

foreach (object o in mySequence) { }

也可以更加明确地声明循环遍历的过程。比如：

foreach (object o in myObject.MySequence()) { }

要实现迭代，必须实现Ienumerable接口，或者实现泛型接口IEnumerable<T> 。假设我们要创建一个字符串集合，如果使用泛型接口，则可以这样定义类：

public class MyEnumerableClass : IEnumerable<string>

{

}

当我们要添加一个迭代器的时候，就可以返回字符串序列。可以利用个数组来存储数据，实现代码是：

private string[] _fruitList = new string[] {

    "Apple", "Banana", "Clementine", "Damson", "Elderberry", "Fig", "Grape",

    "Huckleberry", "Indian Prune", "Jujube", "Kiwi", "Lime"};

public IEnumerator<string> GetEnumerator()

使用yield关键字可以实现迭代器的具体操作。在GetEnumerator方法中，使用yield返回每个数据项。通常情况下，yield返回语句出现在循环体中。

当执行yield返回语句的时候，就会返回相应的值并把状态保存在迭代器中。当下次请求数据的时候，代码会从这个状态继续执行，并继续返回数据。下面的代码将按照次序返回fruits的值，该代码在GetEnumerator方法中实现：

for (int i = 0; i < _fruitList.Length; i++)

{

    yield return _fruitList[i];

}

当执行循环语句的时候，将执行迭代方法中的第一次迭代，在返回第一个结果之前初始化循环语句。当第二次迭代的时候，会重新初始化循环语句。迭代器知道如何继续下次循环。

其中在15行使用了Capacity属性。Capacity 是 ArrayList 可以存储的元素数。Count 是 ArrayList 中实际存储的元素数。

Capacity 总是大于或等于 Count。如果在添加元素时 Count 超过 Capacity，则通过在复制旧元素和添加新元素之前重新分配内部数组来使容量自动增加。

可通过调用 TrimToSize 或通过显式设置 Capacity 属性减少容量。如果 Capacity 的值显式设置，则内部数组也需要重新分配以容纳指定的容量。

        dinosaurs.Add("Amargasaurus");

        dinosaurs.Add("Mamenchisaurus");
