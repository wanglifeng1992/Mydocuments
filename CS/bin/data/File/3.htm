第3章  C# 数据的类型

程序要处理的数据都要有不同的类型，这在数学中也是一样，比如整数、小数、正负数等；要处理数据，首先要指定数据的类型是什么。在C# 中，编写代码的目的也是为了处理数据，包括基本的数学运算，还包括一种特殊的“运算”——字符运算等等，这些运算都要求数据的类型要一致才行。

本章将详细说明C# 中的各种数据的类型，以及这些类型的语言表达方式和使用方式，同时介绍C# 的两个特殊的数据操作：装箱和拆箱。

3.1  数据的类型

计算机是“严谨”的智能系统，凡是它无法“理解”的东西，都会认为无法处理。之所以要预先定义好数据的类型，就是要告诉计算机，要处理的数据是什么样子，以便于它能够正确地运算。

另外，由于计算机存储数据的方式的局限性，也不能进行无限大的数据的计算，定义了数据的类型，同时也就决定了数据存储所使用的空间，以便计算机能合理地分配和使用这些存储控件。比如：整数占用空间小，而小数则要占用更多空间，以便保存小数点后的数据。

3.1.1  数据的类型分类

在C# 中可以把它所能处理的数据的类型划分为两种类型：数据类型和引用类型。具体含义是：

² 数据类型包括一些简单类型（例如，char、int和float）,枚举类型和结构类型。

² 引用类型包括类类型、接口类型、委托(delegate)类型和数组类型。

这两种类型是有区别的，主要在于：

² 数据类型变量直接包含它们的数据，也就是定义了这种数据类型，则就要占用计算机的存储空间来保存数据。

² 引用类型数据则只存储对数据的引用，并不预留存储数据的存储空间。

我们可以用宾馆做例子来说明这两种类型的区别。如果我们拿到房卡，并入住到房间，则类似数据类型，因为房间已经被我们占用；引用类型则只是我们拿到了房卡，我们所具有的只是占用这个房间的“权利”，这相当于引用。在C语言中，这叫做“指针”操作。

对于引用类型，有可能两个变量引用相同的对象，因而可能出现对一个变量的操作影响到其它变量所引用对象的情况。

对于数据类型，每个变量都有它们自己对数据的副本，所以不太可能因为对一个进行操作而影响到其它变量。

如果对C# 数据的类型进行细分，则可以将数据类型归类为以下几种：

² 预定义类型（内置数据类型），如 int 或 char。还有bool，float等。多数预定义类型的数据与数学中的定义具有等同的含义。

² 用户定义数据类型，如 class 或 interface。这是从数据的角度来理解“类”和“接口”，后续章节我们将详细讲述。

在有些资料与编程语言中，也会把数据类型划分为：值类型（用于存储值）和引用类型（用于存储对实际数据的引用）。这是与前面我们提到的数据类型和引用类型相对应的，只是另外一种称呼。比如：在VB.NET 中，数据类型就分为值类型和引用类型。

3.1.2  预定义类型

C# 提供了一系列预定义类型，其中大多数类型对于 C 和 C++ 程序员来说都是比较熟悉的。这些预先定义好的数据类型可以直接使用，无需再次声明。

预定义引用类型是对象和字符串。对象类型是所有其它类型的最根本的基础类型，而字符串类型则还要说明是否是Unicode字符串数据格式。（注：Unicode是国际统一编码的简称，目的是解决不同自然语言在计算机中的编码规则。）

预定义数据类型包括有符号和无符号整数类型、浮点数类型、二进制、字符和十进制类型。具体说明如下：

² 有符号整数类型有sbyte、short、int和long；无符号整数类型有byte、ushort、uint和ulong；而浮点类型有float和double。

² 二进制类型用来表示二进制数据。值或者是真或者是假。使用二进制使我们编写代码比较方便些，至少不需要预先转换数据类型。

² Char类型用来说明Unicode字符。某个char类型变量说明一个单16位Unicode字符。

² 十进制类型适合应用在不能接受舍入误差计算中。通常的例子包括商业计算，例如税收计算和货币转换。十进制类型提供了28个有效位。

注意：之所以将整数的符号（正数或负数）也考虑在内，是因为符号也需要占用存储空间。比如：1-2=-1，则计算结果多出来一个“-”号，计算机就要处理如何存储的问题。而对于无符号的类型，则计算机就不考虑符号问题。

3.1.3  关键字

关键字是对 C# 编译器具有特殊意义的预定义保留标识符。类似，汉字中的成语，已经固定了所代表的含义，就不能乱用或者写错。另外一种比喻就是，关键字就是英语单词，编写程序就是用这些单词写作文，当然不能写错单词。

数据类型名（如int、bool等）本身就是关键字，我们在编程中只能使用，不能重新定义它。它们不能在程序中用作我们自己的标识符号，除非它们有一个 @ 前缀。例如，@if 是一个合法的标识符，而 if 不是合法的标识符，因为它是关键字。

3.1.4  初始化值类型

C# 中的局部变量经初始化后方可使用，就是要预先赋予一定的数值。因此，如果像下面这样声明了一个局部变量而未将其初始化：

int myInt;

那么在将其初始化之前，无法使用此变量。也就是，下面的代码将无法正确执行：

x = myInt+1;

可以使用下列语句将其预先初始化，使之包含具体的数值：

 myInt = new int();  // 采用系统默认的初始化方式，myInt 的值此时是0

此语句等效于：

 myInt = 0;         // 赋值为0

当然，可以像下面这样用同一个语句进行声明和初始化：

 int myInt = new int();

 - 或 -[雨林木风1] 

  int myInt = 0;

new 是 C# 的关键字，这里可以直接使用它。使用 new 运算符时，就会根据变量的数据类型，给变量赋以系统规定的默认值。在上例中，整数类型的数据默认值是 0，因此赋给了 myInt。表3-3 是 C# 的各数据类型的默认值表。

3.1.5  数据类型的转换

我们无法保证所有的运算都是相同类型的，比如除法运算，出现小数是不可避免的。那么，就需要对不同的数据类型进行转换，以保证数据类型一致，运算结果正确。

好在 C# 语言不会强迫我们必须把所有的数据类型都定义为一致的。其实，这也是不可能的，毕竟程序是动态运行的，在运行过程中，出现什么样的结果并不是都能预料到的。

因此，C# 中的数据类型提供了自动转换的机制。数据类型之间的转换可以使用强制转换显式进行，但在某些情况下，也允许隐式转换。例如下面的代码：

static void Main(string[] args)

{

    int i = 10;

    float f = 0;

    f = i;  // 这种隐式转换不会丢失数据

    f = 0.5F;

    i = (int) f;  // 这种强制显示转换会丢失数据

}

显式强制转换调用转换运算符，从一种类型转换为另一种类型。如果未定义相应的转换运算符，则强制转换会失败。可以编写自定义转换运算符，在用户定义类型之间进行转换。

下面的程序将 double 强制转换为 int。如不强制转换则该程序不会进行编译。并出现图3-2中的错误信息：

图3-2  错误提示

要纠正这个错误，就要采用显示强制转换，就是：(int)x。

class Test

{

    static void Main()

    {

        double x = 1234.7;

        int a;

        a = (int)x;  // 类型转换

        System.Console.WriteLine(a);

    }

}

输出：

1234

其中，程序运行输出结果是1234，因为a是整数类型，系统自动丢掉小数点，转换成整数，并把值付给变量 a。

注意：这种为了类型一致，而丢掉小数部分的处理方式，虽然在语法上不会有错误，但实际上是计算结果错了。这种错误要靠编程人员自己来发现。

3.1.6  常数

所谓常数（常量），就是一旦定义，就不在发生变化的数。数学中经常有x+a，这里的a在数学中定义为常数。计算机语言也是如此，对常数的定义是：常数是有意义的名称，替代不变的数字或字符串。

顾名思义，常数存储那些在应用程序的整个执行过程中都保持不变的值。通过使用常数，可以极大地提高代码的可读性，并使代码更易于维护。如果代码包含重复出现的值，或者代码包含的某些很难记住或没有明显意义的数字，则可以在代码中使用常数。常见的就是π，它的值很长，我们通常使用常数定义来记忆它，比如：x=3.14159265。

² 创建和使用常数

C# 语言中，预先包含许多预定义的常数。我们还可以使用 const  语句，按照与创建变量名称相同的规则创建您自己的常数。比如：const  int x=100。

² 常数的范围

是指在无需限定常数名称的情况下便可以引用该常数的所有代码的集合。它与在同一位置声明的变量相同。要创建一个存在于特定过程范围内的常数，须在该过程内声明它。要创建一个在整个应用程序中都可用的常数，应在类的声明部分使用 Public 关键字声明它。

注意：虽然常数在某些方面与变量相似，但您不能像处理变量一样修改它们或向它们赋新值。

以下代码定义了一个常量π，用 PI 表示，然后计算圆的面积：

const double PI=3.14159265358979323846264338327950;

double area;

double r;

r = 2;

area = 2*PI*r*r;

3.1.7  枚举类型

枚举提供一种使用成组的相关常数以及将常数值与名称相关联的方便途径。例如，可以为一组与一周中的七天相对应的整数常数声明一个枚举，然后在代码中使用这七天的名称而不是它们的整数值。

可以使用 enum 关键字创建自己的命名常量集。使用这些数据类型可以声明一组名称或其他文本值，用于定义可以赋给某个变量的所有可能值。

例如，如果程序要处理星期中的各天，则可能希望创建一个名为 DayOfWeek 的新类型。然后，可以声明一个 DayOfWeek 类型的新变量，并为它赋值。使用此数据类型可使代码更具可读性，并且还可降低将非法值或意外值赋给该变量的可能性。定义方法是：

 

public enum DayOfWeek

{

    Sunday = 0,

    Monday = 1,

    Tuesday = 2,

    Wednesday = 3,

    Thursday = 4,

    Friday = 5,

    Saturday = 6

}

然后在代码中就可以使用“枚举类型名.枚举值”的方式来访问枚举的数据。比如：DayOfWeek.Friday，可以获得值5，以此类推。

如果需要访问在 enum 数据类型中使用的名称或词，则可以使用 ToString() 方法来实现，如下所示：

DayOfWeek day = DayOfWeek.Wednesday;

System.Console.WriteLine(day.ToString());  // 输出 Wednesday

也可以对枚举类型进行初始化设置，比如：

enum{Sunday=1,Monday=2,Tuesday=3};

3.1.8  变量

变量表示数值、字符串值或类的对象。变量存储的值可能会发生更改，但名称保持不变。变量是字段的一种类型。顾名思义，变量就是“不断随程序的运行而发生变化的”。数学中经常用到变量，这与 C# 中的变量的含义一致。

“变量”直接对应于内存位置。可以使用单个声明语句定义变量。变量可以是“局部变量”，在过程中定义且仅可用于该过程；它也可以是“成员变量”，在模块、类或结构中定义，但不在任何过程中定义。成员变量又称“字段”。

在 C# 中，变量是使用特定数据类型和标签声明的。必须指定变量是一个 int、一个 float、一个 byte、一个 short 还是 20 多种不同数据类型中的任何一种类型。类型可指定应用程序运行时必须分配用于存储值的精确内存量，以及其他信息。C# 语言在将变量从一种类型转换为其他类型时会强制某些规则。

定义变量的方式就是数据类型后加变量名称即可。比如：int x。下面的代码定义了几个变量，它们的数据类型不同：

int answer = 42;

string greeting = "Hello, World!";

double bigNumber = 1e100;

其中的“le100”是数学中的表示双精度数据的科学记数法，代表“1乘以10的100次方”。

注意：变量的名称不能是关键字、数据类型名称或数字，也不能是系统预定义的名称。通常使用字母或者字母与数字的组合，不能以数字开头。有关变量的详细命名约束，可以参考 C# 编码规范方面的资料。

嵌套类型的可访问性取决于它的可访问域，该域是由已声明的成员可访问性和直接包含类型的可访问域这二者共同确定的。但是，嵌套类型的可访问域不能超出包含它的类型的可访问域。

3.2  装箱与拆箱

装箱和拆箱是 C# 提出的特殊的概念，其目的是更加符合面向对象编程的思想。为了让普通的数据类型变量具有对象的特征（后续章节会讲解对象的含义），并可以像对象一样的使用，提出了装箱和拆箱的概念。

装箱和拆箱是一个形象化的比喻，表示把数据对象化就像装箱子和拆开箱子一样。这样更便于我们理解。

使用装箱和拆箱可以把数值类型转化成对象，或者反之；对象要比数值具备更多的特性，因此在需要的时候可以进行这种转换。

3.2.1  装箱和拆箱

我们已经知道，装箱和取消装箱（拆箱）的目的是使值类型能够被视为对象。对值类型装箱将把该值类型打包到 Object 引用类型的一个实例中。取消装箱将从对象中提取值类型。在下面的示例中，整型变量 i 被“装箱”并赋值给对象 o。

int i = 123;

object o = (object) i;  // 装箱

然后，可以对对象 o 取消装箱并将其赋值给整型变量 i：

o = 123;

i = (int) o;  // 拆箱

以下语句对变量 i 隐式应用装箱操作：

object o = i;  // 隐式装箱

此语句的结果是创建对象引用 o并引用 int 类型的值。该值是赋给变量 i 的值的一个副本。

3.2.3  取消装箱转换练习

与装箱过程相反的就是拆箱，要在运行时成功取消装箱值类型，被取消装箱的项必须是对一个对象的引用，该对象是先前通过装箱该值类型的实例创建的。如果对不同类型的值引用进行取消装箱操作，将出现 InvalidCastException 错误。

下面的示例演示无效的取消装箱及引发的 InvalidCastException 错误。使用 try 和 catch 语句（后续章节我们将介绍），在发生错误时显示错误信息。

class TestUnboxing

{

    static void Main()

    {

        int i = 123;

        object o = i;  // 隐式装箱

        try

        {

            int j = (short) o;  // 尝试使用不同类型进行拆箱

            System.Console.WriteLine("拆箱成功.");

        }

        catch (System.InvalidCastException e)

        {

            System.Console.WriteLine("{0} 错误: 拆箱失败.", e.Message);

        }

    }

}

输出：

指定的转换无效。错误：拆箱失败.

将下列语句：

 int j = (short) o;

更改为下面的代码并运行程序：

int j = (int) o;

输出：

拆箱成功.

可见，装箱和拆箱过程中要保持数据类型的一致性，如果数据类型不一致，则无法实现装箱和拆箱。

装箱和拆箱的主要用途是把值类型转化为引用类型，或者反之。这在编写无法确定数据类型的代码操作中，会非常灵活。因为，我们可以不考虑数据类型，直接当做对象使用就可以了。在后续学习过程中，我们会用到这种特性。
