第1章  开发工具指南

开发工具是编写软件的重要工具，要开发软件，就要使用开发工具。理论上，不是用开发工具也可以编写软件，但那将是非常繁琐和复杂的工作。因此，在现代高效软件生产过程中，都非常看重高效的开发工具，用以帮助开发人员编写软件并发现软件的问题。本章介绍的开发工具就是微软公司的 Visual StudioNET（以下简称VSNET），目前版本是 2008 版。它是专门用于编写 NET 程序的，包括 C#、VB、C++和J#等。在后续编程过程中也将使用该工具。

11  Visual StudioNET集成开发环境初识

通常，把具备设计、编码和调试等编程相关的功能的开发工具软件统称为IDE，就是集成开发环境的意思。这里介绍的 Visual StudioNET 也是一套集成开发工具。能够帮助我们实现高效的 C# 编程。在具体使用该工具之前，简单介绍一下三个相关的新概念。它们是：

² 项。它构成项目，如窗体、源文件和类。项也可作为表示文件的解决方案项，适用于整个解决方案，位于解决方案资源管理器的“解决方案项”文件夹中。最终体现是一个磁盘文件。

² 项目。多个项的集合，通常一个项目代表一套程序，具有相对独立的功能。可以理解为一套软件或一套控件。一般来说，我们开发一套程序，使用一个项目即可。最终也是一个磁盘文件形式，对于C#编程的项目来说，文件名通常是以 csproj为后缀的。

² 解决方案。是多个项目的组合，对于复杂的软件系统来说，需要多个相对独立的项目分别完成不同功能，NET 为了便于管理它们，提出了解决方案的概念，并通过解决方案实现多语言、多功能、多层次的开发管理。比如：有两个项目，分别实现用户操作界面和数据库统计分析，则可以设计两个项目，用来完成操作界面和数据统计分析，然后，通过一个解决方案来管理它们。解决方案也对应一个文件，文件名通常是以 sln 为后缀的。

在后续开发中，我们常常通过建立一个新项目来编程，则系统会自动创建一个解决方案文件，这个时候，解决方案中只有一个项目。以后，还可以添加新项目到这个解决方案中。

111  VSNET安装

VSNET是一套集多种开发语言的开发工具，功能也非常强大，因此，软件本身也很大，大约需要3G的磁盘空间。因此，在安装前，请留出这样的空间，特别要注意的是，无论我们把它安装到哪个磁盘，都要在 C 盘预留至少2G空间为好，因为，软件安装过程中，会把许多默认的程序近乎强行地安装到 C 盘，而没有选择余地。如果空间不够，可能会安装失败。具体的环境要求为：

（1）硬件环境

最低配置：CPU 16MHz以上，1G以上的RAM。系统盘上需要40G。 

（2）软件环境

操作系统：Windows Server 2003、Windows XP。

 

如果要开发Web应用程序，操作系统必须安装Internet信息服务(IIS)。可以通过操作系统的“添加/删除程序”中的“添加/删除windows组件”来添加IIS。

要安装VSNET，可以参考安装盘中的资料，也可以按照安装向导来进行安装。当我们把软件光盘放入光驱后，则自动启动安装程序，进入安装向导。或者，我们直接运行安装盘上的setupexe程序，也可以启动安装向导。根据安装向导的提示进行安装即可。

但要注意下面的几个方面：

² 设置正确的安装位置，就是我们的本地硬盘驱动器，建议把软件安装到非C盘上，这样可以节省 C 盘空间。

² 如果只是进行 C# 编程，则开发功能中只选择Visual C# 即可，其它都不要勾选，这样也可以节省许多磁盘空间。

图1-1  VSNET 安装选择

对于其它安装设置来说，都用默认设置即可。当安装完软件后，安装程序将提示是否安装MSDN帮助文档，我们要选择安装这个文档，因为它是帮助我们学习和使用VSNET的重要电子资料库，建议全部安装上，这样可以随时查阅自己需要的技术资料。

图1-2  VSNET 开发工具启动界面

在图中的界面中，可以看到划分了几个区域，分别代表不同的操作功能。这些小窗口区域是为了使用者操作方便而设置的，下面分别进行说明：

113  新建或打开一个项目

使用 VSNET 编写 C# 程序的第一步，是新建一个项目。然后，在项目中添加项，逐步完成程序的编写。下面我们开始编写第一个C#程序，具体过程是（代码位置：光盘\源代码\chapter1）：

（1）选择“文件”菜单。

（2）选择“新建”，选择“项目”，则出现“新建项目”窗口，见图1-3所示。

（3）选择“项目类型”中的“Visual C# ”。

图1-3  新建项目窗口

（4）选择“控制台应用程序”，这是入门学习的程序类型，我们可以从这里开始，逐步掌握C# 编程技术。

（5）在“名称”标签后输入项目名称，默认有一个名称 ConsoleApplication1，对应的也会自动建立一个解决方案文件，名称也是 ConsoleApplication1，这个名称是自动编号的，下次再建立新项目，则名称默认是 ConsoleApplication2，3等等。我们给自己的项目取名为：chapter1，解决方案名称也自动更改为 chapter1。

（6）在“位置”标签后，指定保存的目录。然后，单击“确定”按钮。默认地，“创建解决方案的目录”选项被勾选，则系统会自动创建一个与解决方案名称同名的目录，这里就是 chapter1。

可以通过操作系统的资源管理器看到这个新建的目录，建立的解决方案文件保存在 chapter1中，建立的其它文件保存在自动建立的chapter1的子目录 chapter1中，包括项目文件和自动创建的第一个代码文件 Programcs都保存在其中。同时还自动建立了bin、obj和properties目录。

提示：这些都是为了清晰地存储程序文件而自动设置的，不是非要这样做，才能编写程序。

（7）系统自动打开 Programcs 程序代码文件。这是编写 C# 程序的第一个代码文件，VSNET 帮助我们创建好了，我们可以直接在这个文件中编辑代码。图1-4是前面这些操作步骤的最后结果图。

图1-4  自动建立了解决方案和程序文件

在上图中，可以看到，解决方案资源管理器已经自动地打开了解决方案，并打开了其中的一个项目 chapter1，以及该项目中的相关项。这些内容通过树状结构来显示，并具有一定的层次关系，可以通过单击“+”号图标来展开这些项，并看到具体的文件名或目录名。

打开一个已经建立的项目文件的方法则简单得多，如同我们打开磁盘文件一样的方便。可以通过以下几个方法来打开现有的项目或解决方案：

² 从系统的资源管理器中找到建立的解决方案文件，比如刚刚建立的 chapter1sln，双击文件名即可自动启动 VSNET 开发工具来打开它。

² 先启动 VSNET，然后在“最近的项目”列表中单击项目名称 chapter1，也可以打开它。

² 从VSNET的“文件”菜单，选择“打开”，选择“项目/解决方案”，并选择文件名，同样可以打开。

注意：一旦建立了解决方案和项目，就不要在系统的资源管理其中随意修改这些文件的名称或内容，否则将破坏整个项目结构，导致VSNET无法打开它们。

114  解决方案的基本构成

下面说明解决方案的基本构成。在解决方案中，有许多要素，分别代表不同的程序内容，在编程开发中，将用来组织、实现不同的目标。常见的解决方案的组成内容为：

² Bin目录，这是保存可执行程序的地方，将来生成的程序都存放在这里。我们也可以指定别的目录，默认的是这个目录而已。

² Obj目录，这里保存的是程序编译生成的中间目标文件，我们基本不会直接使用或操作它们，是VSNET系统生成程序的过程中使用的。

² Properties目录，这里保存编写的程序的配置信息，默认有一个文件 AssemblyInfocs，记录了项目的属性信息。这些信息也可以修改，但无需直接编辑这个文件，而是选择“项目”菜单的“属性”，会出现一个窗口，然后选择“应用程序”，这个窗口中可以设置项目的属性，见图1-5所示。一般情况下，无需修改这些设置。

² 要打开解决方案中的文件，则在解决方案资源管理器的树状结构中，双击一个文件的文件名。

² 要查看某个文件的属性，则单击一个文件名，并查看VSNET窗口下方的一个小窗口—“属性”窗口，可以看到该文件的详细信息。比如上面操作过程中生成的 Programcs 文件，它的属性窗口内容如图1-6所示。

122  控制台应用程序

如果我们熟悉传统的 DOS 命令状态程序，也就是在windows XP 下，运行 CMD命令后，出现的黑屏系统环境，即控制台。则我们就不难理解控制台应用程序的含义，实际上就是命令提示符方式的程序。传统的C语言编写的程序都是这种方式运行的程序。图1-7是控制台应用程序的环境。

图1-7  控制台下运行程序

图中，运行了windows自身原有的控制台程序（也叫做命令程序）netstat，列出当前网络端口的状态，通常用来查看网络端口是否被占用。这个命令程序在windows安装目录的system32下，文件名是：netstatexe。

124  Web 应用程序

ASPNET 提供了一个统一的 Web 开发技术，也就是我们熟悉的互联网上的各种程序，比如：论坛、博客、电子商务等。

其中包括生成企业级 Web 应用程序所必需的各种服务。ASPNET 是 NET Framework 的组成部分，使您能够充分利用公共语言运行库 (CLR) 的功能，如类型安全、继承、语言互操作性和版本控制。

当我们使用 Visual Studio 创建 ASPNET 网站时，实际上就是在使用集成开发环境 (IDE) 的一部分，也称为 Visual Web Developer。Visual Web Developer 与 Visual C# 不同；它拥有自己的设计器，用于在网页上创建用户界面，还拥有进行 Web 开发和网站管理的其他工具。但当您在 C# 中创建 Web 控件的代码隐藏页时，就是在使用 C# 代码编辑器，并且可以像在 Visual C# 中那样，在 Visual Web Developer 中使用编辑器的所有功能。 

13  NET Framework

NET Framework 是所有 VSNET 开发的程序的核心，当我们编写完程序，并生成了可执行文件（比如 exe），可执行文件的运行，需要 NET Framework的支持。NET Framework实质上是一套含有大量系统命令对象的软件，可以单独下载安装。当我们安装 VSNET 的时候，已经自动安装了这套软件。 

131  NET Framework 实现目标

微软公司对NET Framework的定义为：NET Framework 是支持生成和运行下一代应用程序和 XML Web services 的内部 Windows 组件。NET Framework 旨在实现下列目标：

² 提供一个一致的面向对象的编程环境，无论对象代码是在本地存储和执行，还是在本地执行但在 Internet 上分布，或者是在远程执行的。 

² 提供一个将软件部署和版本控制冲突最小化的代码执行环境。

² 提供一个可提高代码（包括由未知的或不完全受信任的第三方创建的代码）执行安全性的代码执行环境。

² 提供一个可消除脚本或解释环境性能问题的代码执行环境。

² 使开发人员的经验在面对类型大不相同的应用程序（如基于 Windows 的应用程序和基于 Web 的应用程序）时保持一致。 

² 按照工业标准生成所有通信， 以确保基于 NET Framework 的代码可与其他代码集成。

132  NET Framework 的主要组件

NET Framework 具有两个主要组件：公共语言运行库和 NET Framework 类库。公共语言运行库是 NET Framework 的基础。可以将运行库看作一个在执行时管理代码的代理，它提供内存管理、线程管理和远程处理等核心服务，并且还强制实施严格的类型安全以及可提高安全性和可靠性的其他形式的代码准确性。 

NET Framework 的另一个主要组件是类库，它是一个综合性的面向对象的可重用类型集合，可以使用它开发多种应用程序，这些应用程序包括传统的命令行或图形用户界面 (GUI) 应用程序，也包括基于 ASPNET 所提供的最新创新的应用程序（如 Web 窗体和 XML ）

133  公共语言运行库的优势

公共语言运行库管理内存、线程执行、代码执行、代码安全验证、编译以及其他系统服务。这些功能是在公共语言运行库上运行的托管代码所固有的。

至于安全性，取决于包括托管组件的来源（如 Internet、企业网络或本地计算机）在内的一些因素，托管组件被赋予不同程度的信任。这意味着即使用在同一活动应用程序中，托管组件既可能能够执行文件访问操作、注册表访问操作或其他须小心使用的功能，也可能不能够执行这些功能。[10] 

1 运行库强制实施代码访问安全。

例如，用户可以相信嵌入在网页中的可执行文件能够在屏幕上播放动画或唱歌，但不能访问他们的个人数据、文件系统或网络。这样，运行库的安全性功能就使通过 Internet 部署的合法软件能够具有特别丰富的功能。[11] 

2 通过实现称为通用类型系统 (CTS) 的严格类型验证和代码验证基础结构来加强代码可靠性。

CTS 确保所有托管代码都是可以自我描述的。各种 Microsoft 和第三方语言编译器生成符合 CTS 的托管代码。这意味着托管代码可在严格实施类型保真和类型安全的同时使用其他托管类型和实例。[12] 

3 运行库还提高了开发人员的工作效率。

程序员可以用他们选择的开发语言编写应用程序，却仍能充分利用其[13] 他开发人员用其他语言编写的运行库、类库和组件。以 NET Framework 为目标的语言编译器使得用该语言编写的现有代码可以使用 NET Framework 的功能，这大大减轻了现有应用程序迁移过程的工作负担[14] 。

4 运行库旨在增强性能。

尽管公共语言运行库提供许多标准运行库服务，但是它从不解释托管代码。一种称为实时 (JIT) 编译的功能使所有托管代码能够以它在其上执行的系统的本机语言运行。[15] 同时，内存管理器排除了出现零碎内存的可能性，并增大了内存引用区域以进一步提高性能。

此外，运行库的托管环境还消除了许多常见的软件问题。例如：运行库自动处理对象布局并管理对对象的引用，在不再使用它们时将它们释放。这种自动内存管理解决了两个最常见的应用程序错误：内存泄漏和无效内存引用。

尽管运行库是为未来的软件设计的，但是它也支持现在和以前的软件。托管和非托管代码之间的互操作性使开发人员能够继续使用所需的 COM 组件和 DLL。

最后，运行库可由高性能的服务器端应用程序（如 Microsoft® SQL Server™ 和 Internet 信息服务 (IIS)）承载。此基础结构使您在享受支持运行库宿主的行业最佳企业服务器的优越性能的同时，能够使用托管代码编写业务逻辑。[16] 

134  NET Framework 类库

NET Framework 类库是一个与公共语言运行库紧密集成的可重用的类型集合。该类库是面向对象的，并提供您自己的托管代码可从中导出功能的类型[17] 。这不但使 NET Framework 类型易于使用，而且还减少了学习 NET Framework 的新功能所需要的时间。此外，第三方组件可与 NET Framework 中的类无缝集成。

例如，NET Framework 集合类实现一组可用于开发您自己的集合类的接口。您的集合类将与 NET Framework 中的类无缝地混合。

正如我们对面向对象的类库所希望的那样，NET Framework 类型使我们能够完成一系列常见编程任务（包括诸如字符串管理、数据收集、数据库连接以及文件访问等任务）。除这些常见任务之外，类库还包括支持多种专用开发方案的类型。例如，可使用 NET Framework 开发下列类型的应用程序和服务：

² 控制台应用程序。

² Windows GUI 应用程序（Windows 窗体）。

² ASPNET 应用程序。

² XML Web services。

² Windows 服务。

例如，Windows 窗体类是一组综合性的可重用的类型，它们大大简化了 Windows GUI 的开发。如果要编写 ASPNET Web 窗体应用程序，可使用 Web 窗体类。

135  客户端应用程序开发

客户端应用程序在基于 Windows 的编程中最接近于传统格的应用程序。这些是在桌面上显示窗口或窗体从而使用户能够执行任务的应用程序类型。客户端应用程序包括诸如字处理程序和电子表格等应用程序，还包括自定义的业务应用程序（如数据输入工具、报告工具等等）。客户端应用程序通常使用窗口、菜单、按钮和其他 GUI 元素，并且它们可能访问本地资源（如文件系统）和外围设备（如打印机）。

另一种客户端应用程序是作为网页通过 Internet 部署的传统 ActiveX 控件（现在被托管 Windows 窗体控件所替代）。此应用程序非常类似于其他客户端应用程序：它在本机执行，可以访问本地资源，并包含图形元素。

以前开发人员结合使用 C/C++ 与 Microsoft 基础类 (MFC) 或应用程序快速开发 (RAD) 环境（如 Microsoft® Visual Basic®）来创建此类应用程序。NET Framework 将这些现有产品的特点合并到了单个且一致的开发环境中，该环境大大简化了客户端应用程序的开发。

包含在 NET Framework 中的 Windows 窗体类旨在用于 GUI 开发。您可以轻松创建具有适应多变的商业需求所需的灵活性的命令窗口、按钮、菜单、工具栏和其他屏幕元素。

例如，NET Framework 提供简单的属性以调整与窗体相关联的可视属性。某些情况下，基础操作系统不支持直接更改这些属性，而在这些情况下，NET Framework 将自动重新创建窗体。这是 NET Framework 集成开发人员接口从而使编码更简单更一致的许多方法之一。

14  IDE 的基本使用方法

编写并修改文本和代码的能力是集成开发环境 (IDE) 的核心功能。Visual Studio 提供许多为特定文件类型定制的编辑器。大多数编辑器都有两个视图：设计器视图和代码视图。所有编辑器都共享一组核心功能，而且还提供特定于正在处理的文件的功能。

141  代码编写

如果打开 Visual C# Windows 窗体进行编辑，则在设计器视图中将打开 C# 窗体编辑器作为此文件的默认编辑器。如果打开 Visual Basic Windows 窗体，则在设计器视图中将打开 Visual Basic 窗体编辑器作为此文件的默认编辑器。这两个编辑器看起来很像，但实际上它们是两个不同的编辑器。利用本主题可熟悉 Visual Studio 中的编辑器的特点和功能。这些编辑器都用到以下术语：

² 编辑器 。一个一般性术语，指 IDE 中可用的所有代码、文本和标记语言编辑器。在过去的版本中，使用的术语是“代码和文本编辑器”。

² 代码视图。一种编辑器模式，在其中可以修改文件中的文本和代码。有时称为“源视图”。所有编辑器都有代码视图。

² 指示器边距。代码视图中编辑器左边缘的灰色长条区域，用来显示调试、书签和错误图标。

² 设计器视图。一种编辑器模式，在其中可以创建应用程序的部分布局和可视外观。并不是所有编辑器都有设计器视图。

² 设计图面。一个一般性术语，指可在其中添加和移动控件与其他元素的可视区域。适用于 Windows 窗体、HTML 页、ASPX 页，以及在编辑时包含设计器视图的任何其他文件。

² 标志符号。 一个一般性术语，指显示在指示器边距中的各种图标，例如断点、书签等。根据所使用的编辑器视图，以及正在编辑的文件类型，编辑器中的工具和功能是不同的。

² 文本编辑器。

² 语句结束。

² 编辑器工具提示。

如果对这些区域中的任何一个进行更改，则将影响 Visual Studio 中的所有编辑器。
