第8章  类的成员

本章将详细讲解C#中类的构成成分，通常叫做类的成员（Member）。这些成员包括我们熟悉的常量、变量、数组、方法函数等。当然，对于C#来说，由于是标准的面向对象编程语言，还有许多其它独特的类成员，本章将逐一介绍。

8.1  类的组成概述

类的成员或者是静态成员 (static member)，或者是实例成员 (instance member)。静态成员属于类，实例成员属于对象（类的实例）。任何一个类都是由这些成员组成的。表8-1提供了类所能包含的成员的说明。

表8-1  类的组成成员清单。

成    员

说    明

常量

与类关联的常数值

字段

类的变量

方法

类可执行的计算和操作函数

属性

与读写类的私有成员相关联的操作

索引器

与以数组方式索引类的实例相关联的操作

事件

可由类生成的通知

运算符

类所支持的转换和表达式运算符

构造函数

析构函数

嵌套类型

类所声明的嵌套类型

注意：在C#中，把组成类的各元素都称作成员；把类也看做一种类型，因此，当提及成员的时候就是指本节中介绍的各个元素；当提到类型的时候有时也指的是类。

实例成员就是我们前面各章节中使用类及成员，基本是实例成员方式。这里不再重复介绍，本节将详细介绍静态类和静态成员。

8.2  静态类和静态成员

静态类和成员用来实现无需创建类的实例就能够访问的数据和函数。静态类成员可用于分离独立于任何对象标识的数据和行为：无论对象发生什么更改，这些数据和函数都不会随之变化。当类中没有依赖于对象标识的数据或行为时，就可以使用静态类。

8.1.1  静态类

静态类用 static 修饰符来声明。这种静态的类不能使用 new 关键字创建实例。静态类的主要特点如下：

² 它们仅包含静态成员。

² 它们不能被实例化。

² 它们是密封的，不可继承。

² 它们不能包含构造函数。

静态类和成员的优点很类似常量，对于编译器来说可以快速编译并无需生成复杂的关联代码，这样会提高运算速度。编写代码也简单得多，因为无需使用new来创建类的实例即可直接使用这些成员。

静态类只能包含静态成员，无论是字段还是方法，并且不能被继承。下面的代码是一个典型的静态类，其中定义了静态字段和方法。

【例8-1-1】静态类。

（代码位置：光盘\源代码\chapter8\staticclass）

    static class staticClass

    {

        private static string name = "ABCDEF";

        public static void Fun()

        {

            Console.WriteLine("Fun 的输出...");

        }

        public static void MoreFun()

        {

            Console.WriteLine("MoreFun 的输出...");

        }

    }

由于是静态类，我们不能创建它的实例，可以直接使用类名。

        static void Main(string[] args)

        {

            MyClass.Fun();

            MyClass.MoreFun();

            Console.ReadLine();

        }

静态类通常用于无需实例化或者只需要初始化一次的情况。我们可以把一些常用的功能编写成静态类。

8.2.2  静态成员

静态成员就是那些无需创建实例也可以访问的类的成员。这些成员包括属性、字段、方法和事件等类成员。静态成员的特点如下：

² 不能使用实例来访问静态成员。

² 只存在静态字段和事件的一个副本。

² 静态方法和属性只能访问静态字段和静态事件。

² 静态成员通常用于表示不会随对象状态而变化的数据或计算；例如，数学库可能包含用于计算正弦和余弦的静态方法。

² 静态方法可以被重载，但不能被重写。

静态成员直接属于类而不是类的对象，因此静态成员不是类的实例的一部分，无法通过类的实例来调用。

注意：索引器不能作为静态成员存在。

下面我们分别举例说明类的静态成员的特性。

1. 静态字段

静态字段也是使用static关键字来定义。如：

class MyClass

 {

  public static int x;

  public static int y = 20;

 }

静态字段可以设置初始值。如果没有初始值则使用它们的默认值。以下示例说明了静态字段的用法。注意，静态字段可以在非静态类中定义。

    //测试静态字段

2. 静态方法

静态方法也可以叫做静态函数。静态方法只能访问静态成员。要访问非静态成员，只能通过类的实例来完成。可以通过类名来调用静态成员。以下代码实现了一个静态方法，并使用类名来调用它。

8.3  构造函数

构造函数也称构造器，是为了对类进行初始化的，类似设置类的默认值一样，因此叫做“构造”。通过构造函数就可以对类进行具体化的实现，也就是实例化，这样，一个类就成了具有特定内容的对象，以后我们就可以使用它。构造函数的名称与类名相同。

8.3.1  基本实现方法

任何时候，只要创建类或结构，就会自动调用它的构造函数。类或结构可能有多个接受不同参数的构造函数。构造函数使得程序员可设置类中成员的默认值、限制实例化以及编写灵活且便于阅读的代码。

如果没有为对象提供构造函数，则默认情况下 C# 将自动创建一个构造函数，该构造函数实例化对象，并将所有成员变量设置为它们的默认值即C# 规定的默认值。静态类和结构也可以有构造函数。

除非类是 static 的，否则 C# 编译器将为无构造函数的类提供一个公共的默认构造函数，以便该类可以实例化。但是，如果将构造函数设置为私有（private）构造函数，则阻止了类被实例化。如以下代码所示：

class test

{

    // 私有构造函数

    private test() { }

    public static double e = 2.71828;

}

当生成项目的时候，VS.NET将提示一个错误：test()不可访问，因为它受保护级别限制。因此，通常不会使用private修饰符来限定构造函数。

结构类型的构造函数与类的构造函数类似，但是不能包含显式默认构造函数，因为编译器将自动提供一个构造函数。此构造函数将结构中的每个字段初始化为默认值表中显示的默认值。

然而，只有当结构用 new 实例化时，才会调用此默认构造函数。例如，下面的代码使用int（Int32）的默认构造函数，因此可以断定整数已初始化：

int i = new int();

Console.WriteLine(i);

然而，下面的代码却导致了编译器错误，因为它没有使用 new创建实例并初始化，而且试图使用尚未初始化的对象：

int i;

Console.WriteLine(i);

从广义上讲，构造函数是对象被创建的时候而执行的方法。通常情况下，我们可以把一些初始化代码放入构造函数。下面的代码是基本的构造函数结构。

public class mySampleClass

{

    public mySampleClass()

    {

        // 这里可以放置代码。

    }

    // 这里是其它类成员的代码。

}

当实例化对象（使用new关键字）的时候，这个构造函数就会自动执行。就像我们调用类的方法一样：

mySampleClass obj = new mySampleClass();

这里使用new关键字创建一个类的实例，这个时候构造函数就已经执行了。

8.3.2  构造函数重载

C# 支持构造函数的重载，也就是说，我们可以编写多个不同的重载构造函数。这与方法重载是一样的。比如以下代码有多个重载的构造函数：

public class mySampleClass

{

    public mySampleClass()

    {

        // 无参数的构造函数。

    }

    public mySampleClass(int Age)

    {

        // 一个参数的构造函数。

    }

    public mySampleClass(int Age, string Name)

    {

        // 两个参数的构造函数。

    }

}

这些构造函数如何被调用，依赖于实例化对象的时候设置了什么参数。参数不同，则自动调用相应的构造函数。比如：

mySampleClass obj = new mySampleClass()

// 调用无参数的构造函数。 

mySampleClass obj = new mySampleClass(12)

// 调用一个参数的构造函数。

8.3.3  从其它构造函数来调用

可以从其它代码位置调用构造函数。比如以下代码：

public class mySampleClass

{

    public mySampleClass(): this(10)

    {

        // 无参数的函数。

    }

    public mySampleClass(int Age)

    {

        //一个参数的函数。

    }

}

注意这里使用了下面代码中的一种语法：

public mySampleClass(): this(10);

这段代码的含义是：this 引用类自身，这样，当我们使用this(10)的时候，实际上执行的是构造函数mySampleClass(int Age) 。这种调用方式叫做初始化器，可以在构造函数中调用另外一个构造函数。

构造函数的执行过程首先是在创建对象实例过程中开始：

mySampleClass obj = new mySampleClass();

然后，就会执行mySampleClass(int Age) ，并先于mySampleClass()中的代码，因为我们在无参数构造函数中使用了以下语法：

public mySampleClass(): this(10)

{

    // 无参数构造函数。

}

以上代码等同于下面的代码的执行过程：

public mySampleClass()

{

    mySampleClass(10)

    // 调用一个参数的构造函数。

}

但要注意，这段代码只是表达程序执行的流程过程，并不能真的这样编写代码，因为编译器无法理解这种用法，不允许这样调用构造函数。

另外，这里只允许使用this和base关键字来调用初始化器，其它方法一律无效。

8.3.4  构造函数的继承

我们知道，类之间具有继承关系，那么相应的构造函数也会伴随这种继承而发生一定的调用关系。下面的代码是类的继承：

【例8-3-1】构造函数的继承。

（代码位置：光盘\源代码\chapter8\mybaseclass）

public class myBaseClass

{

    public myBaseClass()

    {

        // 基类构造函数

    }

    public myBaseClass(int Age)

    {

        // 一个参数的构造函数

    }

}

下面定义派生类。

public class myDerivedClass : myBaseClass

{

    public myDerivedClass()

    {

        // 派生类的构造函数

    }

    public myDerivedClass(int Age):base(Age)

    {

        // 派生类的构造函数，一个参数。

    }

}

以上代码的执行过程是，首先要创建类的实例对象：

myDerivedClass obj = new myDerivedClass()

然后，按照以下顺序执行代码：

1. 执行 myBaseClass() 方法。

2. 执行myDerivedClass() 方法。

如果没有使用base(Age)初始化器，那么就执行没有参数的基类的构造函数。下面我们使用参数来创建派生类的实例。

myDerivedClass obj = new myDerivedClass(15)

那么，代码执行顺序将是：

1. 执行myBaseClass(int Age) 方法。

2. 执行myDerivedClass(int Age) 方法。

这里，因为有参数的构造函数中引用了base关键字，因此，创建实例的时候将执行基类的有参数构造函数myBaseClass(int Age)。

8.3.5  私有构造函数

私有构造函数是使用“private”修饰符定义的，我们既不能创建对象也不能继承这样的类。但可以设置一个属于私有构造函数的公共构造函数。这样就可以访问私有构造函数了。比如：

public class myClass

{

    private MyClass()

    {

        Console.WriteLine("无参数构造函数");

    }

//利用下面的公共构造函数来访问私有构造函数，使用this关键字

    public MyClass(int var):this()

    {

        Console.WriteLine("一个参数的构造函数");

    }   

}

然后使用下面的代码创建实例。

MyClass obj = new MyClass(10);

但要注意，如果使用下面的代码直接创建无参数的实例将会出现错误。

MyClass obj = new MyClass();

8.3.6  静态构造函数

静态构造函数将在创建实例的时候首先调用。使用static关键字来定义，下面的代码是静态构造函数的示例：

public class myClass

{

    static myClass()

    {

        // 这里可以编写代码，但只能访问静态成员。

    }

}

静态构造函数有以下特点：

1. 只能有一个静态构造函数。

2. 不能有参数。

3. 只能访问静态成员。

4. 不能有其它访问修饰符。

8.4  析构函数

析构的含义就是释放和撤销。当一个类失去了它的作用（或者作用范围无效），则系统会自动调用编写的析构函数，这样的好处是，我们只管设置好当不使用这个类的时候该做什么，而无需关心什么时候来析构这个类。本节介绍析构函数的编写方法。

析构函数用于析构类的实例。而当一个类不再使用的时候，我们可以“销毁”它，这样的情况就可以通过析构函数来处理。但要注意：

² 不能在结构中定义析构函数。

² 只能对类使用析构函数。

² 一个类只能有一个析构函数。

² 无法继承或重载析构函数。

² 无法调用析构函数，它们是被自动调用的。

² 析构函数既没有修饰符，也没有参数。

程序员无法控制何时调用析构函数，因为这是由.NET系统的垃圾回收器（GC）决定的。垃圾回收器检查是否存在应用程序不再使用的对象。如果垃圾回收器认为某个对象符合析构，则调用析构函数（如果有）并回收用来存储此对象的内存。程序退出时也会调用析构函数。

析构函数的名称格式是在类名称前加一个“~”号。下面的代码创建三个类，这三个类构成了一个继承关系。

² 类 First 是基类，Second 是从 First 派生的

² Third 是从 Second 派生的。

这三个类都有析构函数。在 Main() 中，创建了派生程度最大的类的实例。

注意：程序运行时，这三个类的析构函数将自动被调用，并且是按照从派生程度最大的到派生程度最小的次序调用。

【例8-4】析构函数示例。

（代码位置：光盘\源代码\chapter8\first）

namespace first

{

    class First

    {

        ~First()

        {

            System.Console.WriteLine("调用 First 的析构函数");

        }

    }

//第二个类继承第一个类

    class Second : First

    {

        ~Second()

        {

            System.Console.WriteLine("调用 Second 的析构函数");

        }

    }

//第三个类继承第二个类

    class Third : Second

    {

        ~Third()

        {

            System.Console.WriteLine("调用 Third 的析构函数");

        }

    }

//测试类

    class TestDestructors

    {

        static void Main(string[] args)

        {

            Third t = new Third();

        }

    }

}

8.5  常数

常数就是固定的常量，在类中保持一定的数值不变，这样可以用来存储我们需要的固定数据。比如，圆周率的　∏　作为常数，因为它是固定不变的，然后我们就可以在以后计算中使用它。

类和结构可以将常数声明为成员。常数是在编译时已知并保持不变的值。常数被声明为字段，声明时在字段的类型前面使用 const 关键字。例如：

class CalendarA

{

    public const int Months = 12;

}

在此示例中，常数 Months 将始终为 12，不能更改，即使是该类自身也不能更改它。常数必须属于整型（sbyte、byte、short、ushort、int、uint、long、ulong、char、float、double、decimal、bool 或 string）、枚举或对 null 的引用。

可以同时声明多个相同类型的常数，例如：

class CalendarB

{

    const int months = 12, weeks = 52, days = 365;

}

只要不会造成循环引用，用于初始化成员的一个常数的表达式就可以引用另一个常数。例如：

class CalendarC

{

    const int months = 12;

    const int weeks = 52;

    const int days = 365;

    const double daysPerWeek = days / weeks;

    const double daysPerMonth = days / months;

}

可以用修饰符 public、private、protected、internal 或 protectedinternal来修饰常数。这些访问修饰符决定了类的用户访问该常数的方式。

 

尽管常数不能使用 static 关键字，但可以像访问静态字段一样访问常数。未包含在定义常数的类中的表达式必须使用类名、一个句点和常数名来访问该常数。例如：

int birthstones = Calendar.months;

8.6  字段

字段也是类的成员之一。通过使用字段，我们可以定义类中的变量数据，并在编程中使用它。传统编程中的变量都可以理解为字段。

注意：有些资料中，把字段（fields）翻译为“域”。

比如我们可以把描述人的类中添加上字段，用来代表人的特性：姓名、性别、年龄等。本节介绍字段的使用方法，它是包含在类或结构中的对象或值。字段使类和结构可以封装数据。

通常使用private修饰符来限定字段的访问，当然也可以使用public修饰符。外部类应当通过方法、属性或索引器来间接访问字段。

字段中存储着类要满足其设计所需要的数据。[雨林木风1] 例如，表示日历日期的类可能有三个整数字段：一个表示月份，一个表示日期，还有一个表示年份。在类块中声明字段的方式如下：指定字段的访问级别，然后指定字段的类型，最后指定字段的名称。例如下面的类中定义了几个字段：

    public int day;

访问对象中的字段是通过在对象名称后面依次添加一个句点“.”和该字段的名称来实现的，具体形式为 object.fieldname。例如：

声明字段时可以使用赋值运算符为字段指定一个初始值。例如，若要自动将 7 赋给 month 字段，需要按如下方式声明 month：

注意：字段设定的初始值不能引用其他实例字段。

字段可标记为：public、private、protected、internal 或 protected internal。这些访问修饰符定义类的使用者访问字段的方式。

可以选择将字段声明为 static。这使得调用方在任何时候都能使用字段，即使类没有任何实例。

 

可以将字段声明为 readonly。只读字段只能在初始化期间或在构造函数中赋值。Static readonly 字段非常类似于常数，只不过 C# 编译器不能在编译时访问静态只读字段的值，而只能在运行时访问。

8.9  方法（函数）

方法函数是类中的关键，方法函数也叫做方法、成员函数。前面章节中我们多次使用这个概念，实现类中的各种操作代码。所有类内部的数据处理都是通过方法来实现的，可以把方法简单理解成传统的 c 语言的函数。方法代表类能够处理数据的能力，可以用方法来实现人的类的上班、下班、走路等动作；也可以实现绘图中的绘制圆形、绘制方形等绘图功能。本节介绍类的方法的相关知识。

8.9.1  声明方法

方法（method）是包含一系列语句的代码块。在 C# 中，每个执行指令都是在方法的上下文中完成的。

方法在类或结构中声明，声明时需要使用修饰符指定访问级别、返回值、方法名称以及任何方法参数。方法参数放在括号中，并用逗号隔开。空括号表示方法不需要参数。下面的类包含三个方法，但没有写具体的方法代码：

class Motor

{

    public void StartEngine() { } //方法1，无参数

    public void AddGas(int gallons) { } //方法2，一个参数

    public int Drive(int miles, int speed) { return 0; } //方法3，两个参数，一个返回值

}

调用对象的方法类似于访问字段。在对象名称之后，依次添加句点（“.”）、方法名称和括号。参数在括号内列出，并用逗号（“,”）隔开。因此，可以使用下面的代码来调用 Motor 类的方法：

int x;

moto.StartEngine(); //调用方法1

moto.AddGas(15); //调用方法2

x = moto.Drive(5, 20); //调用方法3，返回值保存在 x 中

使用方法可以让我们实现处理算法、处理逻辑和各种用户操作，并与类的其它成员协作，实现类所要完成的目标。当我们需要使用这些方法的时候，就可以直接调用它们。

下面的例子实现了吧字符串转换成整数数组的方法函数。该方法有一个字符串参数，作为要转换的字符串。

private int[] ToIntArray(string s)

{

int o = s.Length;

int[] iArray = new int[o];

//循环遍历每个数组元素，并截取字符串中的每个字符放到数组中

for(int x = 0; x < s.Length; x++)

{

iArray[x] = Convert.ToInt32(s.Substring(x,1));

}

return iArray;

}

ToIntArray方法以字符串为参数，转换完毕后，以整数数组形式返回处理结果，因此返回值是整形数组。

其中，Substring(startpos,length) 是字符串中的一个方法，从字符串实例检索子字符串。子字符串从指定的字符位置开始且具有指定的长度。本代码中从字符串s中的x位置检索长度是1的字符串，实际上就是每次只截取一个字符。字符串的位置从0开始，每个字符递增1。

8.10  委托

委托的含义与现实生活中的含义类似，比如委托某人办一件事。这里我们只是说明了要办的事情，但如何去办及怎样来办就要靠被委托方自己来想办法。C#中的委托也是类似的含义，在具体技术上，就是定义了一个方法函数，但没有实现方法体的代码，然后依靠被委托的方法来实现。本节介绍如何实现委托。

8.10.1  委托的概念

委托是一种特殊方法。它只是定义了方法名，并没有具体的方法体代码。然后通过委托方式来具体实现。一旦为委托分配了方法，委托将与该方法具有完全相同的行为。委托方法的调用可以像其他任何方法一样，具有参数和返回值。

与委托的签名（由返回类型和参数组成）匹配的任何可访问类或结构中的任何方法都可以分配给该委托。方法可以是静态方法，也可以是实例方法。这样就可以通过编程方式来更改方法调用，还可以向现有类中插入新代码。只要知道委托的签名，便可以分配自己的委托方法。委托具有以下特点：

² 委托类似于 C函数指针，但它们是类型安全的。

² 委托允许将方法作为参数进行传递。

² 委托可用于定义回调方法。

² 委托可以链接在一起；例如，可以对一个事件调用多个方法。

² 方法不必与委托签名完全匹配。

下面的示例声明了一个名为 Del 的委托，该委托可以封装一个采用字符串作为参数并返回 void 的方法。

public delegate void Del(string message);

其中，关键字delegate用来声明委托方法。该委托不返回任何数据，传递的参数是字符串类型。这样就定义了一个“虚”的方法函数，以后我们可以在其它方法中编写具体的处理代码。

委托可以使单一方法也可以是多个方法。多方法的委托通常用在事件处理上，可以利用委托数组来完成。

8.10.2  声明委托

C#中使用delegate来声明委托。单一方法和多种方法的委托的声明是一样的。下面的代码声明了一个委托。

前面提到，委托只是定义了一个方法体，具体实现还要依靠被委托的方法来实现。下面的代码对前文声明的委托进行具体的实现。使用new运算符来创建一个委托的方法。

这段代码表示，当执行方法d1()的时候，实际上是委托执行了类t1的TestAbc方法。同样，下面的代码则是执行d2(“xxx”)方法的时候，实际上是委托执行了类Test的TestAbc方法。

还可以使用“+”和“-”运算符实现多重委托。因为多个方法的委托实际上是数组结构，所以可以进行添加和删除的操作。当然，这种“+”和“-”操作已经被C#重载了，专门用来操作委托。例如：

要删除一个委托，可以使用Delegate.Remove方法来删除。这个方法有两个参数，一个是现有的委托，一个是要删除的委托。当然，现有的委托要包含要包含要删除的委托。

另外，可以使用“-”删除委托，比如下面的代码：

//从多个委托中删除委托

d1 = d1 - d2; //使用-操作

d1 -= d3; //使用 -= 操作

8.11  事件

如果我们编写过Visual Basic 程序，一定知道事件的基本含义，就是对外部操作的响应。比如，鼠标、键盘等设备的操作。这些是事件的一种情况，对于C#来说，事件已经被提高到消息与响应的关系，不单纯是对设备的响应。本节介绍C#中的事件。

8.11.1  事件的概念

任何事物都与一些事件发生，比如人可能跌到，汽车可能被撞车，而程序也是一样，可能有外部其他程序引发的事件，常见的就是键盘和鼠标操作，这都属于计算机引发的事件；

事件是类在发生其关注的事情时用来提供通知的一种方式。例如，封装用户界面控件的类可以定义一个在用户单击该控件时发生的事件。控件类不关心单击按钮时发生了什么，但它需要告知派生类单击事件已发生。然后，派生类可选择如何响应。

事件具有以下特点：

² 事件是类用来通知对象需要执行某种操作的方式。

² 尽管事件在其他时候（如信号状态更改）也很有用，事件还是用在图形用户界面中使用的比较广泛。

² 事件通常使用委托事件处理程序进行声明。

² 事件可以调用匿名方法来替代委托方法。

事件经常与委托一起使用，通过委托方法，把事件处理委托给相应的方法函数。因此，我们知道委托技术之后，对于事件的编程就会清除得多。

8.12  接口

我们调用的许多外部系统类库都是通过接口来实现的，接口可以扩展一个类的功能，接口类似“接头”的作用，实现除了继承之外的另外一种类与类之间的相互作用关系。接口定义类所能实现的属性、方法和事件。

接口允许将功能定义为一些紧密相关的属性、方法和事件的小组；这样就减少了兼容性问题，因为可以在不损害现有代码的情况下开发接口的增强型实现。在任何时候都可以通过开发附加接口和实现来添加新的功能[雨林木风2] 。本节介绍接口的编程方法。

和类一样，接口也定义了一系列属性、方法和事件。但与类不同的是，接口并不提供实现。它们由类来实现，并从类中被定义为单独的实体。接口表示一种约定，实现接口的类必须严格按其定义来实现接口的每个方面。

类和结构可以像类继承基类或结构一样从接口继承，但有两个例外： 类或结构可继承多个接口；当类或结构继承接口时，它继承成员定义但不继承实现。接口具有下列特性：

² 接口类似于抽象基类：继承接口的任何非抽象类型都必须实现接口的所有成员。[雨林木风3] 

² 不能直接实例化接口。

² 接口可以包含事件、索引器、方法和属性。

² 接口不包含方法的实现。

² 类和结构可从多个接口继承。

² 接口自身可从多个接口继承。

接口是使用 interface 关键字定义。例如：

 

interface IComparable

{

    int CompareTo(object obj);

}

类和结构可以像类继承基类或结构一样从接口继承，但有两个例外：

² 类或结构可继承多个接口。

² 当类或结构继承接口时，它继承成员定义但不继承实现。

8.13  索引器

索引器主要用来像访问数组一样来访问类或者结构对象，这样就可以很方便地把多个相同对象当作数组来访问。索引器类似于属性，不同之处在于它们的访问器采用参数。本节介绍索引器的使用方法。

索引器使得对象可按照与数组相似的方法进行索引。它有以下特性：

² get 访问器返回值。set 访问器分配值。

² this 关键字用于定义索引器。

² value 关键字用于定义由 set 索引器分配的值。

² 索引器不必根据整数值进行索引，由您决定如何定义特定的查找机制。

² 索引器可被重载。

² 索引器可以有多个形参，例如当访问二维数组时。

索引器允许我们按照与数组相同的方式对类、结构或接口进行索引。例如，假定具有一个名为 TempRecord 的类，此类表示在 24 小时内的 10 个不同时间记录的华氏度。此类包含一个表示温度的 float 类型的名为“temps”的数组和表示记录温度的日期的 DateTime。该类的实例为tr。通过在此类中实现一个索引器，客户端可以通过 float temp = tr[4] 而不是 float temp = tr.temps[4] 语法访问 TempRecord 实例中的温度。索引器表示法不仅简化了客户端应用程序的语法，还使其他开发人员能够更加直观地理解类及其用途。

要声明类或结构上的索引器，请使用 this 关键字，如下例所示：

<修饰符> <返回类型> this [参数列表]

 {

  get

  {

   // 返回代码

  }

  set

  {

   // 设置值代码

  }

 }
