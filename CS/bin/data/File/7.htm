第7章  对象、类和结构

本章介绍C#的面向对象编程技术。面向对象编程技术是当今主流的技术，许多编程语言已经逐步采用了这种编程思想。如java、C++等。面向对象的思想也逐步扩展到了其它技术领域，比如数据库系统、软件工程技术等。本章将围绕面向对象中的几个关键技术进行说明。其中包括一些基本的概念、面向对象编程的基本的方法以及C#实现面向对象编程思想的语言特性。

7.1  基本概念

本节首先介绍与面向对象编程相关的一些概念，这些概念不同于结构化编程方法，是需要我们重新认识和理解的理论基础。理解了这些概念，我们才能逐步掌握后续的面向对象的编程技术。

7.1.1  类和对象

首先要说明的是什么是“面向对象(Object Oriented，简称OO)”。面向对象的思想实际上就是根据客观事物的特性来抽象、描述事物及其关系。目前为止，还没有非常明确的定义能够概括面向对象的本质含义。

“面向对象编程”是专指在程序设计中采用封装、继承、抽象等设计方法进行程序设计。面向对象编程技术中，核心的概念是对象（Object）和类（Class），而本章提到的“结构”是一种特殊对象；要掌握面向对象编程，就要理解这些概念。

1. 对象

简单地说，一切都是对象，比如：人、车、思想等。但在程序设计林谷，则对象是包含数据和操作该数据的方法的结构。几乎在 .NET 中执行的所有操作都与对象关联。或者说，都是用来操作对象的。

2. 类

类是对象的符号表示形式。C# 中的类也被看做是一种类型。与建筑图纸是描述建筑物特性一样，类以同样的方式描述组成对象的属性、字段、方法和事件。我们通过编程语言中的类的技术手段来描述抽象的对象。

如果使用蓝图类比，类是蓝图，对象就是基于该蓝图的建筑。通常情况下，更改一个对象中的数据不会更改任何其他对象中的数据。

“类”和“对象”这两个术语有时互换使用，但实际上，类描述对象的“结构”，而对象是类的可用“实例”。每个实例都是其类的一个精确而又不同的副本。由于对象是类的实例，所以创建对象的操作称为“实例化”。如同数学中的数学公式是类，一旦具有了具体的数值，则就成了对象。

C#使用类和结构来实现对象（如 Windows 窗体、用户界面控件和数据结构等）。典型的 C# 应用程序由程序员定义的类和 .NET Framework 的类组成（微软提供了大量现成的类库程序）。

3. 结构

结构可视为轻量类，是创建用于存储少量数据的数据类型的理想选择，不能表示以后可能要通过继承进行扩展的类型。

4. 对象、类和结构的特点

对象、类和结构具有以下特点：

对象是给定数据类型的实例。

对象是具有数据、行为和标识的编程结构。对象数据包含在对象的字段、属性和事件中，对象行为则由对象的方法和接口定义。

对象具有标识，数据集相同的两个对象不一定是同一对象。

7.1.2  类的组成结构

对于C# 来说，类由字段、属性、方法和事件等组成（其它语言不一定如此），它们的具体含义是：

字段和属性表示对象包含的信息。字段类似于变量，因为可以直接读取或设置它们。例如，如果有一个名为“汽车”的对象，则可以在名为“颜色”的字段中存储其颜色。

属性的检索和设置方法与字段类似，但是属性是使用 property Get 和 property Set 过程实现的，这些过程对如何设置或返回值提供更多的控制。在存储值和使用此值的过程之间的间接层帮助隔离数据，并使您得以在分配或检索值之前验证这些值。

方法表示对象可执行的操作。例如，“汽车”对象可以有“启动”、“行进”和“停止”的操作方法。通过向类中添加过程（例程或函数）来定义方法。

事件是对象从其他对象或应用程序接收的通知，或者是对象传输到其他对象或应用程序的通知。事件使对象得以在每当特定情况发生时执行操作。“汽车”类的一个事件示例是“熄火”事件。因为 Windows 是事件驱动的操作系统，所以事件可来自其他对象、应用程序或用户输入（如鼠标单击或按键）。

7.1.3  面相对象的特性

在面向对象的思想中，有几个特性是必须具备的，无论是C# 还是JAVA，或者其它基于面向对象思想的技术。这就如同数学，只要是数学领域的，就要有数学的特性，而不能是语文学科的特性一样。面向对象的特性有：

1. 抽象、封装、继承和多态

抽象、封装、继承和多态是面向对象的基本概念，说明了类自身的特点、类之间的关系。具体解释为：

“抽象”（Abstraction）就是提炼，对于数学来说，抽象的结果是数学公式，对于面向对象编程来说，抽象是本质，抽象的结果就是类。经过对客观对象的抽象，就可以形成编程中的类结构。C# 中，抽象也是一种定义：抽象类和接口（Interface）。抽象的形象体现就是典型的数学公式：y=f(x)。

“封装”（Encapsulation）意味着将一组相关属性、方法和其他成员视为一个单元或对象。对象可以控制更改属性和执行方法的方式。例如，对象在允许属性更改前可验证值。通过隐藏对象的实现细节（一种称为“数据隐藏”的做法），封装还使在以后对实现进行更改更为容易。图7-1说明了自然界不同类的动物和植物的封装性。

“继承”（Inheritance）描述基于现有类创建新类的能力。新类继承基本类的所有属性、方法和事件，而且可用其他属性和方法自定义该新类。而对于基本类（基类）来说，则是派生了这个新类。例如，可基于“汽车”类创建名为“卡车”的新类。“卡车”类从“汽车”类继承“颜色”属性，而且可有其他属性，如“四轮驱动”。图7-2说明了类的继承关系，以家族为例。

      

             图7-1  封装示意图                     图7-2  继承（遗传）示意图

“多态性”（Polymorphism）意味着可以有多个可互换使用的类，即使每个类以不同方式实现相同属性或方法。多态性是面向对象编程的精华，因为它允许使用同名的项，而不管此时在使用什么类型的对象。例如，假设基类“汽车”，多态性使程序员能够为任意数量的派生出的类定义不同的“启动”方法。图7-3以人类为例，说明多态性的含义。

2. 重载、重写、隐藏和实例

重载、重写和隐藏是很容易混淆的类似概念。虽然所有这三种技术都可以让我们创建同名的成员，但它们之间有一些重要的差异。

重载（Overloads ）的成员用于提供属性或方法的不同版本，这些版本具有相同名称但是接受不同数量的参数或者接受不同数据类型的参数。

重写（Override）的属性和方法用于替换在派生类中不适合的继承的属性或方法。重写的成员必须接受同一数据类型和参数数量。派生类继承重写的成员。

隐藏（Hide）的成员用于局部替换具有更广范围的成员。任何类型都可隐藏任何其他类型。例如，可声明隐藏同名继承方法的属性。无法继承隐藏的成员。

实例（instance）就是实际的例子的意思，对于面向对象编程来说，使用 new 操作声明一个对象（类型是一个类），新创建出来的对象就被看做是类的实例。比如人类的实例就是我们自身，人类是抽象的“人”的定义（类），具体化到我们自身，就变成实实在在的例子，所以叫“实例”，实例具有了具体的特性描述内容；而类则是抽象的表述，没有具体化。这是他们的差异。

注意：本节的内容中，有几个英文单词，对应了相关的概念，这些单词将来会成为编程中的关键字，并可直接用于代码编写，且含义不变，因此在这里预先标出。

7.2  类的实现方法

本节介绍使用C# 实现面向对象编程的具体方法，包括关键字、语法结构、编码规则等。既然类是程序设计中用来描述对象的技术手段，那么C# 中就要有对应的语法和语义形式来实现这种抽象描述。

7.2.2  复杂的类的实现

上文曾经提到，C# 中的类通常由常数、字段、方法、属性、索引、事件、操作符、构造函数、析构函数和嵌套类声明等组成。它们的含义是：

常数，就是通常的常数声明，属于类的成员。

字段，实际上就是变量，属于类的成员变量。

方法，就是我们所理解的函数，但要比函数复杂，也可以叫做成员函数。

属性，本质上也是一种方法，主要用来操作私有的字段成员，通常叫做geter和seter。可以控制对字段的访问，也可以对字段进行特定的处理后提供给外部成员访问。

索引，或者叫做索引器，类似属性，但可以利用索引器象访问数组一样来访问类。

事件，事件是对象发送的消息，以发信号通知操作的发生。这些消息可以是我们熟悉的鼠标键盘等设备发出的，也可以是代码逻辑触发的。

操作符，各种运算符，可以进行各种操作。

构造函数，是一种特殊的方法，对类进行默认设置和操作。创建类的新实例的时候，自动调用这个函数；如果没有定义这个函数，则编译器会帮助生成一个。

析构函数，与构造函数相反，在类的实例被释放或撤销的时候执行，通常用来对操作进行最后的处理。默认地，编译器也会创建一个。

嵌套类，可以在类中定义另外一个类，这就是嵌套类。

以上是有关类的组成的各种要素的解释，在后续章节中我们将逐步详细说明它们的技术原理和编程方法。下面将通过一个示例，介绍类的字段、属性和方法的编写方法。

7.3  封装、继承、多态和抽象的实现

本节讲述面向对象的基本概念是如何在C# 中具体实现的。实际上，每个概念都对应了C#中的一套对象和相关的语法结构，都是可以通过编程的方式来完成的。许多类的设计和编码也都是基于这些面向对象的理论和技术来实现的。

7.3.1  类的封装

7.3.2  类的继承

继承是类之间的一种关系，我们已经介绍了继承的概念，可以看出继承是面向对象编程中广泛使用的技术。

可以通过创建一个从现有类派生的新类来扩展现有类的功能，或者说，可以基于现有类定义一个新类，并能够继承现有类的特性。此派生类继承基类的属性，而且可以根据需要添加或重写方法及属性。但结构不能使用继承。

在 C# 中，继承及接口实现均由“:”运算符定义，在类声明中，基类应始终位于最左侧。例如：

public class A

{

    public A() { }//构造器

}

public class B : A //继承 A，或者说由 A 派生出 B

{

    public B() { }//构造器

}

7.3.3  类的多态

前文我们介绍过多态性，通过继承，一个类可以用作多种类型：可以用作它自己的类型、任何基类型，或者在实现接口时用作任何接口类型，这称为多态性。显然，多态性体现了一个类的不同的形态（类型）。典型的多态就是绘图中的点，比如它是一个基类，那么他可以派生出多种图形：线、圆、三角形等。如图7-10所示。

多态性可以允许我们通过基类的引用来调用派生类的方法。这对于多个对象调用多个方法来说就方便许多，因为不要求必须是同样的类型。以下几个示例分别说明类的多态性的编程方式。

在具体编码之前，需要介绍一个概念：虚方法或叫虚函数或虚拟成员。使用virtual关键字来声明。实际上就是一个类的框架，类的属性、方法等可以由继承它的派生类重新设计。这样就非常灵活，每个派生类可以根据自身的需要来重新定义方法的含义和作用。

7.3.6  重载

面相对象另外一个重要概念是“重载”。简单地说，重载就是在一个类中有多个同名不同参数的方法的实现。虽然方法名相同，但因为参数相同，则可以看做是不同的方法。

另外，在C#中，操作符也可以被重载，比如我们熟悉的“+、-”等。实际上，.NET 类库中许多方法都是有多个重载的，当我们输入一个方法的时候，VS.NET的智能感知功能将自动显示所有重载的方法成员，供我们选择，见图7-14是WriteLine方法的重载信息。

图7-14  重载信息

图中说明，WriteLine方法有许多种用法，虽然方法名称都一样，但参数不一样，从而实现的功能也不一样，这就是典型的重载。

1. 方法重载

方法重载就是相同名称的方法有多种不同的功能。每种功能所采用的参数不同，这包括数量或类型不同。对于编译器来说，都是可以其别的。下面的代码说明了对方法Function 的重载，两个方法名字相同，但参数类型不同。

【例7-3-12】方法重载示例。

（代码位置：光盘\源代码\chapter7\overloadA）

using System;

namespace overloadA

{

    class Program

    {

        static void Function(int x) //方法1：int 参数

        {

            x = 12;

            Console.WriteLine(x + " 是整数。");

        }

        //定义一个相同名字的方法，参数名称也相同，但参数类型不同，这是重载的方法

        static void Function(long x)  //方法2：long 参数

        {

            Console.WriteLine(x + " 是长整数。");

        }

        static void Main()

        {

            int a = 12;

            long b = 12;

            Function(a);  //传递整数参数

            Function(b);  //传递长整数参数

            Console.Read();//暂停操作，等待按键退出程序

        }

    }

}

下面的代码实现了另外一种重载，就是采用相同的参数类型，但是参数数目不同。这样，对于C#来说，也认为是不同的方法。

using System;

class Program

{

    static void Function(int x)  //一个参数

    {

        x = 12;

        Console.WriteLine(x + " is an int");

    }

    static void Function(int x, int y)  //两个参数

    {

        Console.WriteLine(x + " 和 " + y + " 都是整数参数");

    }

    static void Main()

    {

        int a = 12;

        int b = 12;

        int c = 12;

        Function(a);  //传递一个参数，调用一个参数的方法

        Function(b, c);  //传递两个参数，则调用两个参数的方法

        Console.Read();

    }

}

与此类似，还可以对类的构造函数进行重载，这样在类初始化的时候，就可以传递不同的参数来执行不同的操作。

在C#中，使用this关键字来代表类自身。那么就可以对类进行参数传递，然后传递给重载的构造方法函数。下面的代码重载构造函数，使类可以接受参数。

2. 运算符（操作符）重载

前文提到C#的标准运算符也可以重载，我们可以定义自己的运算符，改变系统默认的运算符操作的方式。但在重载操作符的时候，需要注意以下几点：

重载操作符必须是公共、静态访问标识。

不能返回void类型。

至少有一种运算操作是定义的类类型。比如加法重载，最终在重载方法中要使用系统的标准运算操作“加”。

相关联的运算符也会被重载。比如重载了“==”，则也重载了“!=”运算符。

下面的代码实现了对两个类的相加，实际上是对类的字段的加法操作，并返回计算结果。本来系统是不允许两个类直接相加的，但通过对运算操作符的重载，则可以实现两个类的加法运算。

【例7-3-13】操作符重载。

（代码位置：光盘\源代码\chapter7\overloadB）

using System;

namespace overloadB

{

    class myClass

    {

        private int myValue = 0;

        //重载“+”操作符，使之可以实现两个类的相加

        public static myClass operator +(myClass left, myClass right)

        {

            myClass tmpClass = new myClass();

            //把类的值属性相加并返回类的实例

            tmpClass.value = left.value + right.value;

            return (tmpClass);

        }

        //显示类的值

        public void Display()

        {

            System.Console.WriteLine("字段 myValue 的值 ={0}\n", myValue);

        }

        //定义类的属性，用来存取myValue 数据

        public int value

        {

            set

            {

                myValue = value;

            }

            get

            {

                return (myValue);

            }

        }

    }

    //对代码进行加法测试

    class Example

    {

        public static void Main()

        {

            myClass newClass;

            myClass oldClass = new myClass();//新的实例

            myClass dummy = new myClass();//另外一个实例

            dummy.value = 2;//设置属性

            oldClass.value = 3;//设置属性

            newClass = dummy + oldClass;//对两个实例进行相加，这时将使用重载的加法操作运算

            newClass.Display();//通过方法显示结果

            System.Console.WriteLine("属性 newClass.value 的值 ={0}\n", newClass.value);//通过属性显示结果，属性值已经变化

            Console.ReadLine();

        }

    }

}

为了能够实现复杂的运算，我们还可以为结构类型、数组、字符串等设计自己的重载操作，这样就可以利用系统的运算符来实现各种运算，而无须使用许多代码来完成。比如整型数组的操作，要把两个数组的每个元素对应地求和相加，通常使用循环实现，如果设计一个“+”重载方法，则可以简单许多。

7.3.7  重写

在前面的示例中，我们不断使用了重写这个词，并在代码中编写了方法的重写代码。本节总结性地介绍一下有关重写的技术。

重写修饰符主要用于扩展或者修改抽象类和虚成员（方法、属性、索引器或事件）的具体功能。比如下面的代码中，通过重写计算面积的方法来实现抽象类ShapesClass的虚方法Area()，并实现面积的计算。

abstract class ShapesClass

{

    abstract public int Area();//计算面积的抽象方法

}

class Square : ShapesClass

{

    int x, y;

    // 因为ShapesClass 中的Area是抽象方法，这里需要重写该方法，以便实现面积计算

    public override int Area()

    {

        return x * y;

    }

}

7.3.8  隐藏

方法隐藏是就是方法函数对基类有效，而对于派生类，则无效。如果派生类希望有同样的方法，那么就要定义自身的同等功能的方法，名称可以相同。以下程序代码说明了方法隐藏的含义。

首先，我们编写一个基类BC和派生类DC，基类有一个输出信息的方法Display()，而派生类没有。

class BC

{

  public void Display()

  {

     System.Console.WriteLine("基类显示的信息。");

  }

}

//派生类DC

class DC : BC

{

}

class Demo

{

  public static void Main()

  {

     DC obj = new DC();

     obj.Display();//调用基类方法

  }

}

输出结果：基类显示的信息。

这段代码可以正常运行，因为基类提供了public 方法Display()。派生类可以继承这个方法，并可以调用它。

下面的代码则在派生类DC中添加了自己的Display()方法，并与基类的方法同名，但输出的内容不同（这个并不是至关重要的）。

class BC

{

  public void Display()

  {

     System.Console.WriteLine("基类显示的信息。");

  }

}

class DC : BC

{

  public void Display()//派生类自己的方法

  {

     System.Console.WriteLine("派生类显示的信息。");

  }  }

class Demo

{

  public static void Main()

  {

     DC obj = new DC();

     obj.Display();//调用自己的方法，而不是基类的

  }

}

输出结果：派生类显示的信息。

程序正常运行了，但在编辑状态，实际上有一个系统警告信息。见图7-17所示。

图7-17  隐藏方法

警告信息提示我们，基类成员Display()被隐藏，无法发挥作用，如果是希望隐藏，可以使用new关键字来明示，如果不使用new，编译器就会觉得很含糊。下面的代码进行了必要的修正，使用new关键字来隐藏方法，则警告信息立即消失。

class BC

{

  public void Display()

  {

     System.Console.WriteLine("BC::Display");

  }

}

class DC : BC

{

  new public void Display()//使用new关键字明示隐藏

  {

     System.Console.WriteLine("派生类显示的信息。");

  }

{

  public static void Main()

  {

     DC obj = new DC();

  }   }

输出结果：派生类显示的信息。

方法的隐藏可以是多级的，最后起作用的方法是最后一个派生类所定义的方法，但不要忘记使用new关键字来声明。

7.4  结构

结构作为一种特殊的类也可以具备面向对象的特征，但并不是所有特征。利用结构这种特殊的类型可以保存具备一定结构特性的数据，比如个人简历：姓名、性别、年龄、学历等组成了一个整体，形成一个结构化的数据。比如定义一个坐标点（x,y），可以用结构来描述，并可以进行赋值使用。

结构是使用 struct 关键字定义的，结构具有以下特点：

结构是值类型，而类是引用类型。

向方法传递结构时，结构是通过传值方式传递的，而不是作为引用传递的。

与类不同，结构的实例化可以不使用 new 运算符。

结构可以声明构造函数，但它们必须带参数。

一个结构不能从另一个结构或类继承，而且不能作为一个类的基类。

所有结构都直接继承自 System.ValueType，后者继承自 System.Object。

结构可以实现接口。

在结构中初始化实例字段是错误的。

struct 类型适于表示 Point、Rectangle 和 Color 等轻量对象。尽管可以将一个点表示为类，但在某些情况下，使用结构更有效。例如，如果声明一个 1000 个 Point 对象组成的数组，为了引用每个对象，则需分配更多内存；这种情况下，使用结构可以节约资源。由于 .NET Framework 包含名为 Point 的对象，因此我们可以定义结构“CoOrds”来描述一个点的信息：

public struct CoOrds

{

    public int x, y;

    public CoOrds(int p1, int p2)//构造器

    {

        x = p1;

        y = p2;

    }

}

上面的代码中，定义了一个结构CoOrds，同时定义了结构的构造器（构造函数），用来给结构成员变量x和y赋值。但要注意：

构造函数必须有参数。

成员变量，即字段不能有初值。

如果使用 new 运算符创建结构对象，则会创建该结构对象，并调用适当的构造函数。与类不同，结构的实例化可以不使用 new 运算符。如果不使用 new，则在初始化所有字段之前，字段都保持未赋值状态且对象不可用。

对于结构对象，不能像类那样存在继承关系。一个结构不能从另一个结构或类继承，而且不能作为一个类的基类存在。但是，结构从基类 Object 继承。结构可实现接口，其方式同类完全一样。[雨林木风1] 

我们可以使用结构创建值类型的对象，就像int、float、bool等类型一样。结构类型与它们相比，则有更大的灵活性。比如数据具备一定的关联性，需要成组出现，使用数组是无法实现的，或者比较繁琐；使用基本类型则根本无法实现，那么就应该使用结构。

下面是结构的示例代码，实现了一个简单的坐标系统，使用结构来记录坐标位置：x和y，并可以对坐标进行赋值操作。

【例7-4】使用结构。
