第4章  运算符、表达式和语句

本章介绍C# 语言的运算符、语句和表达式。正如自然语言一样，要表达自己的想法，就要通过符合语法要求的方式来进行。而对于计算机语言来说，则最基本的就是计算，还有由这些运算符号及变量组合成的表达式，另外，还包括代表特定处理方式的语句。这很类似自然语言中的单词、计算公式和句子。只不过，在C# 中要按照C# 规定的语法规则来书写这些句子而已。

4.1  运算符和表达式

对于运算符，我们并不陌生。因为在数学中，我们经常使用运算符来计算。C# 提供大量运算符，这些运算符是指定在表达式中执行哪些操作的符号。例如 ==、!=、<、>、<=、>=、+、-、^、&、|、~、++、-- 和 sizeof()。但是，这些运算符已经不只是停留在数学计算的范畴，更是扩展了它们所代表的本来含义。

这些运算符可以组合、嵌套，并最终构成代表特定运算规则的表达式。在有些编程语言中，把这些符号称为“操作符（operator）”，因此，操作符和运算符的意思是一样的。

4.1.1  基本运算符

基本运算符提供了一些常规的数据操作方式。在C# 中提供了针对类成员访问、类型转换、数组定义等等的各种运算方式。基本运算符包括的类别是：

² 数据类型操作类：typeof，checked，unchecked，用于数据类型的操作。

² 基本运算类：x++，x--，用于数值型的操作。

4.1.2  一元运算符

所谓一元运算就是只有一个变量的操作运算，比如：+x。最简单的一元操作就是正数和负数，对于数学来说是一种表达方式；对于 C# 来说，认为是一种运算操作。一元操作符也是常用的操作符，并且不仅仅是针对数值型数据，对字符串数据也可以操作。

另外一元操作符也可以进行两个变量的二元操作运算。比如：运算符“-”，即可以表示负数，也可以用来实现两个数的减法操作。同样，运算符“+”，既可以表示正数（通常不用在代码中书写），也可以表示两个数相加，还可以表示两个字符串的联接。

4.1.4  移位运算符

移位运算的本质是对二进制编码进行位置的移动。任何数据在计算机系统中的最终表现形式都是二进制的，因此，可以进行移位运算。“<<”移位运算的基本规则是：

² 如果第一个操作数是 int 或 uint（32 位数），则移位数由第二个操作数的低 5 位给出。

² 如果第一个操作数是 long 或 ulong（64 位数），则移位数由第二个操作数的低 6 位给出。

² 第一个操作数的高序位被放弃，低序空位用 0 填充。移位操作从不导致溢出。

² “>>”移位运算的规则是：

² 如果第一个操作数为 int 或 uint（32 位数），则移位数由第二个操作数的低五位给出（第二个操作数 & 0x1f）。

² 如果第一个操作数为 long 或 ulong（64 位数），则移位数由第二个操作数的低六位给出（第二个操作数 & 0x3f）。

² 如果第一个操作数为 int 或 long，则右移位是算术移位（高序空位设置为符号位）。如果第一个操作数为 uint 或 ulong 类型，则右移位是逻辑移位（高位填充 0）。

移位操作中，移动的数字是二进制表示方式的数字。比如：1左移1位就是10。那么，如果我们要把1变为10，则无需采用加法，而是直接移位，这种方法速度很快。这只是体现移位运算优势的一种，在图形图像处理中，应用更为广泛。

² bool，布尔类型。

² true，表示真。

² false，表示假。

比如：true & true，结果是“真”；true & false，结果是“假”等等。

二元 ^ 运算符是为整型和 bool 类型预定义的。对于整型，^ 将计算操作数的按位“异或”。

条件运算符 (?:) 根据布尔型表达式的值返回两个值中的一个。

格式：condition ? first_expression : second_expression;

逻辑运算可以细分为以下两种运算操作方式，它们之间有所不同：

² 逻辑类运算：&、|和^是当操作数是整型的时候，将按二进制位来进行逻辑预算。比如：10&9，转换为二进制则是：1010&1001，如果按位进行“与”操作，结果是：1000，即十进制的8。如果是bool逻辑数值，则直接进行逻辑运算。比如：true&false 结果是 false。

² 条件类运算：&&和||，实际上是代表“并且”和“或者”的逻辑关系，只能对bool值进行运算，不适合其它数据类型。比如：1&&2是错误的，而true&&false是正确的。

在大量的程序编码中，经常用到这些逻辑运算操作，后续讲解的循环、判断语句，将与这些逻辑运算有紧密的关系。下面的代码测试了基本的逻辑运算：

Console.WriteLine(10&9);

Console.WriteLine(10^9);

Console.WriteLine((1>2)&&(2>3));

Console.ReadLine();

输出结果是：

8

3

False

另外一种有趣的逻辑运算是“?:”，表示三个操作数的逻辑判断。具体语法规则是：

condition ? first_expression : second_expression;

如果条件condition 为 true，则计算第一表达式first_expression并以它的计算结果为运算结果；如果为 false，则计算第二表达式second_expression并以它的计算结果为运算结果。但是只计算两个表达式中的一个。比如：1<2?0:1，运算结果是0，因为1肯定小于2。

使用条件运算符，可以更简洁、雅观地表达那些否则可能要求 if-else 结构的计算。[雨林木风2]

左移赋值运算符。x<<=y 等同于 x=x<<y

>>=

右移赋值运算符。x>>=y 等同于 x=x>>y

??

如果 ?? 运算符的左操作数非 null，该运算符将返回左操作数，否则返回右操作数。y=x ?? 1 表示如果 x 为 null 则 y=1

4.1.8  算术溢出问题

在数学上，算数溢出也是常见的现象，我们通常使用“四舍五入”解决问题，这种溢出往往是数字位数不够造成的。如果不限制数字的长度，那么一个简单的π值，就可以无限长，如果让计算机存储，恐怕也存储不了。

而在 C# 中，由于各种数据类型已经限定了它的存储长度，自然也就限制了这种类型的变量所能容纳的数据的值的长度。比如，int 类型的长度是4字节，能存储的最大长度是2,147,483,647。如果超过这个长度，就要尝试使用 long 类型。

算术运算符（+、-、*、/）产生的结果可能会超出涉及的数值类型可能值的范围。而一般情况下会出现以下情况：

² 整数算术溢出或者引发 OverflowException，或者丢弃结果的最高有效位。整数被零除总是引发 DivideByZeroException。

² 浮点算术溢出或被零除从不引发异常，因为浮点类型基于 IEEE 754，因此可以表示无穷和 NaN（不是数字）。

² 小数算术溢出总是引发 OverflowException。小数被零除总是引发 DivideByZero Exception。

当发生整数溢出时，产生的结果取决于执行上下文，该上下文可为 checked 或 unchecked（可见这两个操作符是很有用的）。

在 checked 上下文中引发 OverflowException。在未选中的上下文中，放弃结果的最高有效位并继续执行。因此，C# 使我们有机会选择处理或忽略溢出。

除算术运算符以外，整型之间的强制转换也会导致溢出（例如，将 long 强制转换为 int）并受 checked 或 unchecked 执行的限制。然而，按位运算符和移位运算符永远不会导致溢出。

4.2  表达式

表达式是可以计算且结果为单个值、对象、方法或命名空间的代码片段。表达式可以包含文本值、方法调用、运算符及其操作数，或简单名称。简单名称可以是变量、类型成员、方法参数、命名空间或类型的名称。具体含义是：

² 表达式类由常量、变量和运算符等组成。比如：x=x+y是加法运算表达式。

² 表达式的运算符标明在操作数上运用了哪种操作。运算符的例子包括+、-、*、/和new。操作数的例子包括字面值、字段、局部变量和表达式。

² 当表达式包括多个运算符时，运算符的优先级（precedence）控制各个运算符执行的顺序。例如，表达式x+y*z将以x+(y*z)的形式计算，原因就是运算符“*”的优先级高于运算符“+”。

最简单的两种表达式类型是文本和简单名称。文本是没有名称的常数值。例如，在下面的代码示例中，5 和 "Hello World" 都是文本值：

int i = 5;

string s = "Hello World";

在上面的示例中，i 和 s 都是用于标识局部变量的简单名称。在表达式中使用这些变量时，变量值检索后将用于表达式中。

总之，表达式可以是值、变量、公式等任何可以计算并产生计算结果的符号和操作符的组合。

表达式中的运算符按照称为运算符优先级的特定顺序计算。下表根据运算符执行的操作类型将它们划分到不同的类别中。类别按优先级顺序列出。

当表达式中出现两个具有相同优先级的运算符时，它们根据结合性进行计算。左结合运算符按从左到右的顺序计算。例如，x * y / z 计算为 (x * y) / z。右结合运算符按从右到左的顺序计算。

赋值运算符和三元运算符 (?:) 是右结合运算符。其他所有二元运算符都是左结合运算符。然而，C# 标准没有指定何时执行表达式中的增量指令的“设置”部分。例如，下面的代码示例的输出为6：

int num1 = 5;

num1++;

System.Console.WriteLine(num1);

而下面的代码示例的输出却是未定义的：

int num2 = 5;

num2 = num2++;  //不推荐

System.Console.WriteLine(num2);

因此，建议不要使用后一种方式。可以在表达式两侧使用括号来强制在计算其他任何表达式之前计算该表达式。例如，2 + 3 * 2 正常情况下计算为 8。这是因为乘法运算符的优先级高于加法运算符。将该表达式写为 (2 + 3 ) * 2 的形式，结果将是 10，因为它指示 C# 编译器必须在计算乘法运算符 * 之前计算加法运算符 +。

4.4  选择语句

选择语句可以控制代码执行流程，为了有条件地处理一些事情，我们往往要根据设定的条件来选择如何做，这在程序里也常常如此，通常利用条件语句和开关语句来实现；选择语句根据某个条件是否成立来将程序控制权移交给特定的流。

4.4.1  if 语句

if 语句根据 Boolean 表达式的值选择要执行的语句。下面的示例将 Boolean 标志 flagCheck 设置为 true，然后在 if 语句中检查该标志。输出为：The flag is set to true。

1. bool flagCheck = true;

3. {

4.    Console.WriteLine("The flag is set to true.");

5. }

6. else

7. {

8.     Console.WriteLine("The flag is set to false.");

9. }

如果括号里的表达式计算为 true，则执行 Console.WriteLine("The boolean flag is set to ture."); 语句。执行 if 语句之后，控制传递给下一个语句。在此例中不执行 else 语句。

如果想要执行的语句不止一个，可以通过使用 {} 将多个语句包含在块中，有条件地执行多个语句，如上例所示。在测试条件时执行的语句可以是任何种类的，包括嵌套在原始 if 语句中的另一个 if 语句。在嵌套的 if 语句中，else 子句属于最后一个没有对应的 else 的 if。

选择语句中使用下列关键字：

² if（conditon）{} else{}（如果满足条件则....否则...），如果只有一行表达式，可以省略“{}”。

² switch(var){ case : default:}（当...的时候则...），可以省略default。

选择语句第一步要测试给定的条件，在测试条件时执行的语句可以是任何种类的，包括嵌套在原始 if 语句中的另一个 if 语句。在嵌套的 if 语句中，else 子句属于最后一个没有对应的 else语句的 if。例如：

1.    if (x > 100)

2.    if (y > 200)

3.        Console.Write("Statement_1");

4.    else

5.        Console.Write("Statement_2");

if (x > 100)

{

    if (y > 200) Console.Write("Statement_1");

}

else

    Console.Write("Statement_2");

在此例中，如果条件 (x > 100) 计算为 false，将显示 Statement_2，而（y > 200）则作为（x>100）条件为true的时候要执行的代码。可以看出，大括号决定了流程控制区域的划分。

4.4.2  switch控制语句

switch是一个控制语句，它通过将控制传递给其体内的一个 case 语句来处理多个选择条件。因此，通常叫做“开关”语句。具体结构是：

² switch(x)，x是变量。

² case 条件1:，第一种情况。如果x满足“条件1”的值，则执行“:”号后的语句。

² 依此类推，条件2，条件3等，列举出多种可能。

² default:，表示默认状况下执行的代码，因为我们无法确保罗列的条件都满足。

² break，用来结束一种条件的代码执行，跳出这个条件。

控制传递给与开关的值匹配的 case 语句。switch 语句可以包括任意数目的 case 实例，但是任何两个 case 语句都不能具有相同的值。语句体从选定的语句开始执行，直到 break 将控制传递到 case 体以外。在每一个 case 块（包括上一个块，不论它是 case 语句还是 default 语句）的后面，都必须有一个跳转语句（如 break）。[雨林木风3] 但有一个例外，（与 C++ switch 语句不同）C# 不支持从一个 case 标签显式贯穿到另一个 case 标签。这个例外是当 case 语句中没有代码时。

如果没有任何 case 表达式与开关值匹配，则控制传递给跟在可选 default 标签后的语句。如果没有 default 标签，则控制传递到 switch 以外。

下面编写一个switch 语句的测试练习，用来判断变量的值，当满足某个条件时，输出信息，并结束 switch语句的执行。

【例4-2-3】switch 语句测试。

int caseSwitch = 1;

switch (caseSwitch)

{

    case 1: //如果是1，则执行这里的代码

        Console.WriteLine("Case 1");

        break;

    case 2://如果是2，则执行这里的代码

        Console.WriteLine("Case 2");

        break;

    default://如果既不是1也不是2，则执行这里的代码

        Console.WriteLine("Default case");

        break;

}

输出结果应该是“Case 1”，因为变量的值是1，满足第一个case语句的条件。

注意：控制传递给与开关的值匹配的 case 语句。switch 语句可以包括任意数目的 case 实例，但是任何两个 case 语句都不能具有相同的值。语句体从选定的语句开始执行，直到 break 将控制传递到 case 体以外。在每一个 case 块（包括上一个块，不论它是 case 语句还是 default 语句）的后面，都必须有一个跳转语句（如 break）。[雨林木风4] 

有一种特殊情况，可以用来简化多种可能的条件下，如何控制代码执行过程。下面的示例显示了空 case 标签可以从一个 case 标签贯穿到另一个，依次判断，然后执行最后一次判断的代码并跳出语句。这段代码实现了当n的值是“1、2、3”中的任何一个时，进行统一的处理，那就是 case 3 中的代码段。

【例4-2-4】switch 的变体。

using System;

{

    static void Main()

    {

        int n = 2;

        switch(n)

        {

            case 1:

            case 2:

            case 3:

                Console.WriteLine("可能是 1, 2, 3.");

                break;

        default:

            Console.WriteLine("无法确定是什么值.");

            break;

        }

    }

}

可能是 1, 2, 3.

虽然n的值是2，但因为case语句中没有具体处理代码，也没有break，所以会依次执行下去，直到遇到break为止。

4.5  迭代（循环）语句

迭代或循环技术在编程中经常使用，可以反复地处理数据；通过使用迭代语句可以创建循环。迭代语句导致嵌入语句根据循环终止条件多次执行。除非遇到跳转语句，否则这些语句将按顺序执行。迭代语句中使用下列关键字，基本是英语单词的原意：

² do{statement} while(condition) （循环执行，直到满足条件为止）。

² for （condition）{statement}（循环执行，直到满足条件为止）。

² foreach x in xlist （凡是在特定对象中的都进行处理）。

² while(condition){statement} （当满足条件的时候就执行）。

由于 while 表达式的测试在每次执行循环前发生，因此 while 循环执行零次或更多次。这与执行一次或多次的 do 循环不同。

在循环体内，可以适当使用 break、goto、return 或 throw 语句将控制权转移到循环之外终止该循环。若要将控制权传递给下一次迭代但不退出循环，使用 continue 语句。

下面，我们分别通过示例练习，体验如何使用这些循环语句进行编码，并测试各种循环语句的不同处理效果。

4.5.1  do while 语句

4.5.2  for 循环

for 循环主要用于在一定范围内的循环，在循环的同时进行运算，根据运算结果自动决定是否结束循环。当然，这种循环也可以用其它循环替代。

for 循环重复执行一个语句或语句块，直到指定的表达式计算为 false 值。for 循环对于迭代数组和顺序处理非常方便。在下面的示例中，int i 的值写入控制台，并且 i 在每次通过循环时都加自动 1。

【例4-2-6】for 循环测试。

1. using System;

2. class ForLoopTest

3. {

4.     static void Main()

5.     {

6.         for (int i = 1; i <= 5; i++)//括号中的语句将被执行

7.         {

8.             Console.WriteLine(i);

9.         }

10.     }

11. }

输出：

1

2

3

4

5

for 语句重复执行括起来的语句，这里的语句也决定了循环是否结束。具体执行过程是：

（1）首先，计算变量 i 的初始值。通常都会赋给一个初始值。

（2）然后，当 i 的值小于等于 5 时，条件的计算结果为 true，将执行 Console.WriteLine 语句并重新计算 i。

（3）当 i 大于 5 时，条件变成 false 并且控制传递到循环外部。

由于条件表达式的测试在循环执行之前发生，因此 for 语句执行零次或更多次。for 语句的所有表达式都是可选的；例如，下列语句用于写一个无限循环：

 for (;;)

{

    // 这里的语句永远都要执行下去，除非使用break、goto等结束循环

}

for 循环有一个变体的写法，用来循环处理一组数据，根据数据组中的值，作为循环的条件，并且自动执行下去，无需而外的代码来判断，这就是 foreach。

4.5.3  foreach 语句

foreach 语句为数组或对象集合中的每个元素重复一个嵌入语句组。foreach 语句用于循环访问集合以获取所需信息。嵌入语句为数组或集合中的每个元素继续执行。当为集合中的所有元素完成迭代后，控制传递给 foreach 块之后的下一个语句。

【例4-2-7】foreach 测试，在此示例中，使用 foreach 显示整数数组的内容。有关数组的详细说明，将在后续章节中提到。

4.6  跳转语句

在前述章节中，我们提到了主动结束循环的方法，就是采用跳转语句。特别是当我们需要在特殊的条件下，跳出循环的时候，采用这种方式更加方便。

使用跳转语句执行语句分支，该语句导致立即传递程序控制到其它代码。跳转语句中使用下列关键字：

4.7  异常处理语句

正如汽车会有交通事故，有些时候发生的事情往往是不可预知的，而对于成千上万行的程序代码来说，任何程序都会有异常情况发生，这不仅仅是程序有编写的缺陷，有时候因为计算机环境不同和处理的数据的变化，也会导致不是错误的“异常”情况，我们要通过代码来捕获并控制这些情况，否则程序可能崩溃退出或者导致计算机系统死机。

C# 为处理在程序执行期间可能出现的反常情况（称作异常）提供内置支持。这些异常由正常控制流之外的代码处理。一旦出现异常情况，则程序执行流程就转向异常处理代码，并把控制权交付给这些代码。这样做的好处是，我们可以使用异常处理语句监管程序错误，而不用管这些错误是因为什么发生的。列异常处理包括以下语句：

² throw，人为发出异常信息。在自定义对象中往往使用它来自定义异常。

² try-catch，尝试捕获异常情况，并处理异常。

² try-finally，尝试捕获异常情况，并执行一些代码。finally 中的语句将被执行。

² try-catch-finally，尝试捕获异常情况并处理异常，同时也执行一些代码。

4.7.1  catch 语句

其中，catch 语句中将获得异常信息，通常在这个语句中添加错误处理代码或者信息提示代码。如果不编写任何代码，则程序忽略异常，继续执行。

finally 中的代码是必须执行的代码，也就是，即使catch中有代码，那么执行完毕，还要继续执行 finally中的代码。下面的例子使用了两个 catch 语句，捕获错误信息。

由于我们在catch语句中定义了异常的类别是Exception （异常），因此，系统知道具体的信息是什么，这些信息保存在变量 e中。
